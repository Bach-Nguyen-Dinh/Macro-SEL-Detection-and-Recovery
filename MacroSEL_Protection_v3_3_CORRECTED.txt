1. SYSTEM ARCHITECTURE
================================================================================

PROTECTION STRATEGY:
  One master P-channel MOSFET (high-side, 5V input rail)
  Six independent current-sensing channels (one per downstream rail)
  Any fault on any rail trips all rails simultaneously
  Hardware-only infinite retry: ~1 second off between attempts
  SR latch maintains fault state even when sensing circuits lose power
  OBC monitors device health via telemetry (optional failsafe)

RESPONSE TIME (worst case, Rail 6):
  Difference amplifier:  9.2 µs  (Rail 6, highest gain)
  Comparator:            0.6 µs  (TLV1704-SEP typical propagation delay)
  Logic chain:          <0.01 µs (CMOS propagation, negligible)
  P-FET turn-off:        0.3 µs  (gate RC = 100Ω × 2.5nF)
  TOTAL:                ~10.1 µs worst case

RESPONSE TIME (fastest, Rail 1 or 2):
  Difference amplifier:  1.8 µs
  Rest of chain:         0.9 µs
  TOTAL:                ~2.7 µs

ADEQUACY: Macro-SEL events develop over 10–1000+ µs before causing
permanent damage. Both figures are well within the safe response window
for macro-SEL. Micro-SEL (sub-µs pulse) is not detected, but micro-SEL
on advanced CMOS nodes is typically self-recovering.

3. POWER DOMAIN STRATEGY (CRITICAL FOR AUTONOMOUS RETRY)
================================================================================

TWO INDEPENDENT POWER DOMAINS:

┌─────────────────────────────────────────────────────────────────────┐
│ DOMAIN 1: SENSING (3.3V from Buck #1 output)                       │
│ Purpose: Amplify shunt voltages for comparison                     │
│ Can lose power: YES (during shutdown, no rails to sense anyway)    │
├─────────────────────────────────────────────────────────────────────┤
│ Components:                                                         │
│  - U_AMP_A: LMP7704-SP (op-amp quad, channels 1-4)                 │
│  - U_AMP_B: LMP7704-SP (op-amp quad, channels 5-6)                 │
│                                                                     │
│ CRITICAL: LMP7704-SP MUST use 3.3V supply. At 5V supply, SEE       │
│ vulnerability increases (unqualified voltage). Always power from   │
│ Buck #1 3.3V output.                                                │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│ DOMAIN 2: CONTROL (5V from 5V_IN, before master P-FET)             │
│ Purpose: Maintain fault state, timing, and enable control          │
│ Can lose power: NO (must stay alive to retry after shutdown)       │
├─────────────────────────────────────────────────────────────────────┤
│ Components:                                                         │
│  - U_CMP_A, U_CMP_B: TLV1704-SEP comparators (fault detection)     │
│  - U_OR_A, U_OR_B: SN54SC4T32-SEP OR gates (fault aggregation)     │
│  - U_LATCH: SN54SC4T00-SEP NAND gates (SR latch, fault memory)     │
│  - U_SCH: SN54SC6T17-SEP Schmitt buffers (timers)                  │
│  - U_AND: SN54SC4T08-SEP AND gates (enable logic)                  │
│  - U_INV: SN54SC6T14-SEP inverters (P-FET gate driver)             │
│  - Q_dis: JANTXV2N2222A NPN (timer discharge)                      │
│  - R_timer, C_timer: RC components for retry delay                 │
│  - V_ref dividers (5× voltage dividers for comparator reference)   │
│                                                                     │
│ All TLV1704-SEP and SN54SC ICs rated 2V-6V operation. Running at   │
│ 5V is within spec and does not increase SEE vulnerability.         │
└─────────────────────────────────────────────────────────────────────┘

WHY THIS SEPARATION IS MANDATORY:

The v3.2 design powered everything from Buck #1 3.3V. When a fault shut
off the P-FET:
  1. Buck #1 input cut → 3.3V collapses
  2. All circuits lose power
  3. Timer capacitor discharges
  4. Enable logic dies
  5. System stuck OFF forever (no autonomous restart possible)

With dual domains:
  1. Buck #1 collapses (sensing domain dies)
  2. Control domain STAYS ALIVE (powered from 5V_IN)
  3. SR latch REMEMBERS fault occurred
  4. Timer continues charging
  5. When timer expires, latch resets and enables P-FET
  6. Autonomous restart works ✓

INTERFACE BETWEEN DOMAINS:

Signals crossing from Domain 1 (3.3V) to Domain 2 (5V):
  - V_AMP_1 through V_AMP_6 (op-amp outputs)
  
These are voltage signals (0-2.2V range), compatible with 5V logic inputs.
No level shifting required.

When Buck #1 dies:
  - V_AMP_n outputs go to 0V (or high-Z, pulled to 0V via comparator input)
  - Comparators see V_AMP = 0V < V_ref = 0.996V
  - This would make FAULT_n = HIGH (no fault) if not for the SR latch
  - SR latch already SET (FAULT_LATCHED = HIGH) before Buck #1 died
  - Latch output controls Q_dis and enable logic, not GLOBAL_FAULT
  - System correctly stays off until timer completes

4. MASTER PROTECTION SWITCH
================================================================================

COMPONENT: Q1 – BUP06CP038F-01

Package:     DPAK (TO-252AA) power package
Type:        P-channel power MOSFET
V_DSS:       -55V (comfortably above 5V system)
I_D:         -35A continuous (7× system max current)
I_D peak:    -140A pulsed (handles latchup transients)
R_DS(on):    38 mΩ typ @ Vgs = -4.5V
             44 mΩ max @ Vgs = -4.5V (125°C)

Radiation tolerance:
  TID:       30-50 krad(Si) by design (DMOS structure, thick oxide)
  SEE LET:   46 MeV·cm²/mg (moderate immunity)

POWER DISSIPATION AT RATED LOAD:
  I = 5A, R_DS(on) = 44 mΩ (worst case, hot)
  P = I² × R = 5² × 0.044 = 1.1W
  θ_JA ≈ 50°C/W (minimal heatsinking, TO-252 on 2oz copper)
  ΔT_J = 1.1W × 50 = 55°C
  T_J = 25°C (ambient) + 55°C = 80°C (well within 150°C max)

CONNECTIONS:

           5V_IN ──────────┬─── Source (S)
                           │
                      [Q1 P-FET]
                           │
           5V_OUT ─────────┴─── Drain (D)

  Gate (G) circuit:
    5V_IN ──[R1: 10kΩ]── GATE ──[R2: 100Ω]── GATE_DRIVE
                          │
                        [C2: 100nF to Source]

  R1 (10kΩ pull-up): Ensures P-FET defaults to OFF on power-up or if
                      gate driver loses power. Gate pulled to Source voltage
                      → Vgs = 0V → FET OFF (fail-safe).

  R2 (100Ω series):  Gate drive current limiting and dV/dt control.
                      With C_GS ≈ 2.5nF, τ = 100Ω × 2.5nF = 250ns turn-off.

  C2 (100nF snubber): Reduces gate ringing and dV/dt stress during switching.

GATE DRIVER:
  U_INV gate #1 (SN54SC6T14-SEP Schmitt inverter)
  Input:  ENABLE_GLOBAL (from AND gate)
  Output: GATE_DRIVE

  Logic:
    ENABLE_GLOBAL = HIGH → GATE_DRIVE = LOW  → Vgs = -5V → FET ON
    ENABLE_GLOBAL = LOW  → GATE_DRIVE = HIGH → Vgs = ~0V → FET OFF

  SN54SC6T14-SEP at 5V supply:
    Output LOW:  <0.1V (strong pull to GND)
    Output HIGH: >4.9V (strong pull to 5V)

  When GATE_DRIVE = LOW:
    Gate = (5V × 100Ω + 0V × 10kΩ) / (100Ω + 10kΩ) ≈ 0.05V
    Source = 5V
    Vgs = 0.05V - 5V = -4.95V → FET fully ON (R_DS = 38mΩ)

  When GATE_DRIVE = HIGH (≈5V):
    Gate = 5V, Source = 5V
    Vgs = 0V → FET OFF

TURN-OFF TIME UNDER FAULT:
  From FAULT detected to 5V_OUT dropping:
    Comparator:      0.6 µs
    Logic chain:     0.01 µs
    Gate RC rise:    3 × τ = 3 × 250ns = 0.75 µs (to reach Vgs threshold)
    TOTAL:          ~1.4 µs

  Latchup current × time = 20A × 1.4µs = 28 µA·s
  This is negligible thermal energy, no damage to FET or load.

  Slew rate:           1 V/µs
  Input CM range:      (V-) - 0.2V to (V+) + 0.2V = -0.2V to 3.5V
  Differential input:  ±0.3V ABSOLUTE MAXIMUM (protection resistors mandatory)
  TID:                 100 krad(Si) RLAT
  SEE LET:             85 MeV·cm²/mg (very robust)
  Offset voltage:      ±0.5 mV typical
  Input bias current:  ±50 pA

CIRCUIT PER CHANNEL:

    V_RAIL+ (before shunt) ──[R_p: 1kΩ]─┬──[R1: 9.09kΩ]──┬─── V+
                                         │                 │
                                         └──[R2: varies]───┤
                                                           │
                                                       [OP-AMP]
                                                           │
    V_RAIL- (after shunt)  ──[R_p: 1kΩ]─┬──[R3: 9.09kΩ]──│─── V-

    G = R2 / R1_eff = R2 / 10.09kΩ
    V_AMP = G × (V_RAIL+ - V_RAIL-) = G × V_sense

6. CURRENT SENSE CIRCUITS – STAGE 2: COMPARATORS
================================================================================

COMPARATOR IC: TLV1704-SEP (quad, TSSOP-14)
Supply:        5V from 5V_IN (Domain 2 – control, always powered)
Quantity:      2 packages (8 comparators, 6 used)

TLV1704-SEP specifications:
  Supply voltage:      2.2V – 24V (powered at 5V from 5V_IN)
  Input CM range:      Rail-to-rail (0V to VCC)
  Propagation delay:   560 ns typical, 1.5 µs max
  Input offset:        ±300 µV typ, ±2 mV max
  Output type:         Open drain (requires pull-up resistor)
  TID:                 30 krad(Si) RLAT
  SEE LET:             43 MeV·cm²/mg

WIRING PER CHANNEL:

    V_AMP_n (from op-amp) ────────────────┬──→ IN+ (non-inverting)
                                          │
               V_ref (0.996V) ────────────┴──→ IN- (inverting)
                                          
                                      COMPARATOR
                                          │
                                         OUT (open drain)
                                          │
                              [R_pull: 10kΩ to 5V]
                                          │
                                       FAULT_n

LOGIC:
  V_AMP < V_ref → IN+ < IN- → OUT = LOW  → FAULT_n = LOW (normal)
  V_AMP > V_ref → IN+ > IN- → OUT = Hi-Z → R_pull → FAULT_n = HIGH (FAULT DETECTED)

Active-HIGH fault signaling (clearer convention):
  - FAULT_n = LOW during normal operation (comparator pulling down)
  - FAULT_n = HIGH when fault detected (comparator releases, pull-up pulls high)
  - OR gates: any FAULT_n HIGH → GLOBAL_FAULT HIGH

WHEN BUCK #1 COLLAPSES DURING SHUTDOWN:
  - Op-amps lose power → V_AMP_n → 0V (or high-Z pulled to 0V)
  - 0V < 0.996V → IN+ < IN- → OUT = LOW → FAULT_n = LOW (appears normal)
  - GLOBAL_FAULT = LOW (false "no fault" state)
  - But SR LATCH already SET (FAULT_LATCHED = HIGH) before Buck #1 died
  - Latch output (FAULT_LATCHED) controls enable, not GLOBAL_FAULT directly
  - System correctly stays off until latch is reset by timer

V_REF GENERATION (POWERED FROM 5V_IN, NOT 3.3V):

Target: ~1.0V (matches V_AMP at trip point)
Supply: 5V from 5V_IN (stays stable even when Buck #1 dies)

  5V ──[R_top: 40.2kΩ]── V_ref ──[R_bot: 10.0kΩ]── GND

Calculation:
  V_ref = R_bot / (R_top + R_bot) × 5V
        = 10.0 / (40.2 + 10.0) × 5
        = 10.0 / 50.2 × 5
        = 0.996V (within 0.4% of 1.0V target)

Using E96 0.1% resistors:
  R_top = 40.2kΩ (E96)
  R_bot = 10.0kΩ (E96)

GROUPING (minimize divider count):
  Group A: Rails 1 & 2 share one V_ref divider (identical trip voltage)
  Groups B-E: Rails 3, 4, 5, 6 each individual divider (same values)
  
  Total: 5 divider circuits × 2 resistors = 10 resistors

DECOUPLING:
  One 100nF ceramic (X7R, 0805) capacitor from each V_ref node to GND.
  Placed <2mm from comparator IN+ pin.
  Prevents comparator switching noise from modulating reference voltage.

PULL-UP RESISTORS:
  R_pull = 10kΩ (0805, 1%, standard) to 5V
  One per comparator output (6 total)
  
  Pull-up to 5V (not 3.3V) since comparators are 5V powered.
  FAULT_n signals are 5V logic levels.

OPTIONAL HYSTERESIS:
  Not implemented in baseline design (comparators switch cleanly at threshold).
  If needed: Add R_hyst = 1MΩ from OUT to IN+ (creates ~3mV hysteresis).

PACKAGE ALLOCATION:

  U_CMP_A (TLV1704-SEP package #1):
    Comp 1: Rail 1
    Comp 2: Rail 2
    Comp 3: Rail 3
    Comp 4: Rail 4

  U_CMP_B (TLV1704-SEP package #2):
    Comp 1: Rail 5
    Comp 2: Rail 6
    Comp 3: UNUSED (tie IN+ to GND, IN- to GND, leave OUT no-connect)
    Comp 4: UNUSED (tie IN+ to GND, IN- to GND, leave OUT no-connect)

POWER CONNECTION:
  Both packages: VCC = 5V from 5V_IN, GND = GND
  Bypass: 100nF ceramic <5mm from VCC pin

7. LOGIC CONTROLLER
================================================================================

All logic ICs powered from 5V (5V_IN, Domain 2 – control).
All SN54SC family: TSSOP-14 package, 30 krad TID, 2V-6V operation.

────────────────────────────────────────────────────────────────────────
SECTION A: FAULT AGGREGATION (6-input OR using 2-input gate cascade)
────────────────────────────────────────────────────────────────────────

IC: U_OR_A, U_OR_B – SN54SC4T32-SEP (quad 2-input OR)
Quantity: 2 packages

Purpose: Combine all 6 FAULT_n signals into single GLOBAL_FAULT signal.

Logic: GLOBAL_FAULT = FAULT_1 OR FAULT_2 OR FAULT_3 OR FAULT_4 OR FAULT_5 OR FAULT_6

Implementation (5 gates across 2 ICs):

  U_OR_A (package #1):
    Gate 1: FAULT_1 OR FAULT_2 → F12
    Gate 2: FAULT_3 OR FAULT_4 → F34
    Gate 3: FAULT_5 OR FAULT_6 → F56
    Gate 4: UNUSED (tie both inputs to GND)

  U_OR_B (package #2):
    Gate 1: F12 OR F34 → F1234
    Gate 2: F1234 OR F56 → GLOBAL_FAULT
    Gates 3-4: UNUSED (tie both inputs to GND)

Signal meanings:
  All FAULT_n = HIGH → GLOBAL_FAULT = HIGH (no fault detected)
  Any FAULT_n = LOW  → GLOBAL_FAULT = LOW (fault on that rail)

NOTE: When Buck #1 collapses, all FAULT_n go HIGH (false "no fault").
This is why we need the SR latch – to remember the fault state.

────────────────────────────────────────────────────────────────────────
SECTION B: SR LATCH (NEW IN v3.3 – CRITICAL FOR AUTONOMOUS RETRY)
────────────────────────────────────────────────────────────────────────

IC: U_LATCH – SN54SC4T00-SEP (quad 2-input NAND)
Quantity: 1 package (2 gates used for latch, 2 spare)

Purpose: Remember that a fault occurred, even after Buck #1 collapses and
         GLOBAL_FAULT goes undefined. Hold system off until retry timer
         explicitly resets the latch.

CIRCUIT (classic cross-coupled NAND SR latch):

            ┌──────┐
  SET ──────┤      │
            │ NAND ├────┬──→ Q (FAULT_LATCHED)
      ┌────┤      │    │
      │    └──────┘    │
      │                │
      │    ┌──────┐    │
      └────┤      │    │
           │ NAND ├────┘
  RESET ───┤      │
           └──────┘

Using NAND gates from U_LATCH (SN54SC4T00-SEP):
  Gate 1: NAND(SET_L, Q')    → Q
  Gate 2: NAND(RESET_L, Q)   → Q'
  Gates 3-4: UNUSED (tie both inputs to 5V)

Where:
  Q  = FAULT_LATCHED (output we use)
  Q' = complement (not used directly)

Input preparation:
  SET_L   = NOT(GLOBAL_FAULT)  (active low, so invert GLOBAL_FAULT)
            Normal: GLOBAL_FAULT = LOW → SET_L = HIGH (inactive)
            Fault:  GLOBAL_FAULT = HIGH → SET_L = LOW (active, sets latch)
            
  RESET_L = NOT(RETRY_PULSE)   (active low, pulse from timer)
            Normal: RETRY_PULSE = LOW → RESET_L = HIGH (inactive)
            Retry:  RETRY_PULSE = HIGH → RESET_L = LOW (active, resets latch)

We need 2 inverters for SET_L and RESET_L:
  Use U_INV gates #2 and #3 (SN54SC6T14-SEP hex inverter)
    Gate #2: GLOBAL_FAULT → SET_L
    Gate #3: RETRY_PULSE  → RESET_L

OPERATION:

State 1 – Normal operation:
  GLOBAL_FAULT = LOW (all FAULT_n = LOW, no faults detected)
  SET_L = HIGH (inverted from LOW)
  Latch not triggered
  FAULT_LATCHED = LOW

State 2 – Fault detected:
  GLOBAL_FAULT goes HIGH (any FAULT_n went HIGH)
  SET_L goes LOW (inverted from HIGH)
  Latch SETS
  FAULT_LATCHED = HIGH (and STAYS HIGH even when GLOBAL_FAULT changes)

State 3 – Buck #1 collapses:
  GLOBAL_FAULT goes LOW (false "no fault" because V_AMP → 0V → all FAULT_n LOW)
  SET_L goes HIGH (inverted from LOW)
  Latch HOLDS previous state
  FAULT_LATCHED = HIGH (still remembering fault)

State 4 – Retry timer completes:
  RETRY_PULSE goes HIGH briefly (~10ms)
  RESET_L goes LOW (inverted)
  Latch RESETS
  FAULT_LATCHED = LOW
  System can now restart

This is the CRITICAL missing piece from v3.2. Without this latch, when
Buck #1 collapsed, GLOBAL_FAULT went HIGH and killed the timer. With the
latch, FAULT_LATCHED stays HIGH and keeps system off until timer says retry.

────────────────────────────────────────────────────────────────────────
SECTION C: STARTUP DELAY (~110-225 ms)
────────────────────────────────────────────────────────────────────────

IC: U_SCH gate #1 – SN54SC6T17-SEP (hex Schmitt buffer)
Power: 5V from 5V_IN

Purpose: Prevent false tripping during power-on inrush before rails settle.

CIRCUIT:

  5V ──[R_start: 1.0MΩ]── TIMING_A ──[C_start: 330nF]── GND
                             │
                             └──→ Schmitt buffer input
                                  
                                  Schmitt buffer output → PGOOD_DELAYED

TIMING:
  Capacitor charges from 0V toward 5V through 1.0MΩ resistor.
  When voltage crosses Schmitt VT+, output goes HIGH.

  t = R × C × ln(VCC / (VCC - VT+))

SN54SC6T17-SEP thresholds at VCC = 5V (from datasheet):
  VT+ (positive): min 1.43V, typ 2.00V, max 2.71V
  VT- (negative): min 0.74V, typ 1.06V, max 1.35V

Delay range:
  Min (VT+ = 1.43V): t = 1.0M × 330n × ln(5/(5-1.43)) = 0.33 × 0.326 = 108ms
  Typ (VT+ = 2.00V): t = 1.0M × 330n × ln(5/(5-2.00)) = 0.33 × 0.511 = 169ms
  Max (VT+ = 2.71V): t = 1.0M × 330n × ln(5/(5-2.71)) = 0.33 × 0.781 = 258ms

Range: ~110-260ms (wider than 3.3V version due to 5V supply threshold scaling)

This is adequate for Buck #1 and PMIC startup sequences (typically 50-100ms).
If longer delay needed, increase C_start to 680nF → 220-520ms range.

OPERATION:
  PGOOD_DELAYED = LOW during power-up (cap charging, input < VT+)
  PGOOD_DELAYED = HIGH after delay (cap charged, input > VT+)
  
  Used in enable AND gate to prevent P-FET turn-on until rails settled.

────────────────────────────────────────────────────────────────────────
SECTION D: RETRY TIMER (~0.6-1.35 second)
────────────────────────────────────────────────────────────────────────

IC: U_SCH gate #2 – SN54SC6T17-SEP (same package as startup delay)
Power: 5V from 5V_IN

Purpose: After fault shutdown, hold system off for ~1 second to allow:
  - Latchup current to fully dissipate
  - PMIC and buck converter capacitors to discharge
  - Component temperatures to stabilize

CIRCUIT:

  5V ──[R_timer: 1.0MΩ]── TIMING_B ──[C_timer: 2.2µF]── GND
                             │
                             └──→ Schmitt buffer input
                                  
                                  Schmitt buffer output → RETRY_ACTIVE

  Discharge transistor:
    Q_dis: JANTXV2N2222A NPN (SOT-23)
      Collector → TIMING_B
      Emitter   → GND
      Base      → [R_base: 10kΩ] ← FAULT_LATCHED

CONTROL LOGIC:
  When FAULT_LATCHED = LOW (no fault):
    Q_dis base = LOW → Q_dis OFF → capacitor discharged via R_timer (slow)
    Actually, we want rapid discharge when fault clears.
    
  CORRECTION – Q_dis should be controlled by inverted FAULT_LATCHED:
    When FAULT_LATCHED = LOW (no fault):
      INV(FAULT_LATCHED) = HIGH → Q_dis ON → cap shorted to GND
      TIMING_B = 0V → RETRY_ACTIVE = LOW
      
    When FAULT_LATCHED = HIGH (fault remembered):
      INV(FAULT_LATCHED) = LOW → Q_dis OFF → cap charges
      TIMING_B rises 0V → VT+ over 0.6-1.35s
      When crosses VT+ → RETRY_ACTIVE = HIGH

Use U_INV gate #4 to invert FAULT_LATCHED before driving Q_dis base.

TIMING CALCULATION:
  RC = 1.0MΩ × 2.2µF = 2.2 seconds (time constant)
  
  t = RC × ln(VCC / (VCC - VT+))
  
  At VCC = 5V, using same Schmitt thresholds:
    Min (VT+ = 1.43V): t = 2.2 × ln(5/(5-1.43)) = 2.2 × 0.326 = 0.72s
    Typ (VT+ = 2.00V): t = 2.2 × ln(5/(5-2.00)) = 2.2 × 0.511 = 1.12s
    Max (VT+ = 2.71V): t = 2.2 × ln(5/(5-2.71)) = 2.2 × 0.781 = 1.72s

Range: ~0.7-1.7 seconds (slightly longer than 3.3V version)

This variation is acceptable – any delay >0.5s gives adequate settling time.

RETRY_PULSE GENERATION:
  We need a brief pulse when RETRY_ACTIVE goes HIGH to reset the SR latch.
  
  Simple RC differentiator:
    RETRY_ACTIVE ──[C_pulse: 10nF]── RETRY_PULSE ──[R_pull: 100kΩ to GND]
  
  When RETRY_ACTIVE rises: C couples the edge → brief positive pulse (~1ms)
  When RETRY_ACTIVE falls: negative pulse (ignored by latch reset logic)
  
  RETRY_PULSE width ≈ 5 × R × C = 5 × 100kΩ × 10nF = 5ms

────────────────────────────────────────────────────────────────────────
SECTION E: ENABLE LOGIC (CORRECTED FROM v3.2)
────────────────────────────────────────────────────────────────────────

IC: U_AND – SN54SC4T08-SEP (quad 2-input AND)
Power: 5V from 5V_IN

Purpose: Combine startup delay, fault state, and retry timing to generate
         final P-FET enable signal.

LOGIC EQUATION:
  ENABLE_GLOBAL = PGOOD_DELAYED AND (NOT FAULT_LATCHED)

Note: We don't need RETRY_ACTIVE in the enable logic because FAULT_LATCHED
      already handles it:
        - Fault occurs → FAULT_LATCHED goes HIGH → ENABLE = LOW
        - Timer expires → FAULT_LATCHED reset to LOW → ENABLE = HIGH
      
      RETRY_ACTIVE is only used to generate the reset pulse for the latch.

IMPLEMENTATION:
  Need NOT(FAULT_LATCHED) first:
    Use U_INV gate #5: FAULT_LATCHED → NOT_FAULT_LATCHED

  Then 2-input AND:
    U_AND gate #1:
      Input A: PGOOD_DELAYED
      Input B: NOT_FAULT_LATCHED
      Output:  ENABLE_GLOBAL

  U_AND gates 2-4: UNUSED (tie both inputs to GND)

LOGIC TABLE:

PGOOD_DELAYED  FAULT_LATCHED  NOT_FAULT  ENABLE_GLOBAL  State
─────────────────────────────────────────────────────────────
    LOW             X            X            LOW        Startup settling
    HIGH           HIGH         LOW           LOW        Fault detected, off
    HIGH           LOW          HIGH          HIGH       Normal operation

GATE DRIVER:
  U_INV gate #1: ENABLE_GLOBAL → GATE_DRIVE (inverts for P-FET)
  
  ENABLE_GLOBAL = HIGH → GATE_DRIVE = LOW  → P-FET ON
  ENABLE_GLOBAL = LOW  → GATE_DRIVE = HIGH → P-FET OFF

────────────────────────────────────────────────────────────────────────
INVERTER ALLOCATION SUMMARY (U_INV – SN54SC6T14-SEP)
────────────────────────────────────────────────────────────────────────

U_INV has 6 inverter gates. Usage:

  Gate #1: ENABLE_GLOBAL       → GATE_DRIVE (P-FET driver)
  Gate #2: GLOBAL_FAULT        → SET_L (SR latch SET input)
  Gate #3: RETRY_PULSE         → RESET_L (SR latch RESET input)
  Gate #4: FAULT_LATCHED       → controls Q_dis base (timer discharge)
  Gate #5: FAULT_LATCHED       → NOT_FAULT_LATCHED (for enable AND gate)
  Gate #6: UNUSED (tie input to GND)

Note: Gates #4 and #5 have same input (FAULT_LATCHED) but serve different
      purposes. This is intentional – gate #4 drives Q_dis base (needs
      current drive), gate #5 feeds AND gate (logic level only).

────────────────────────────────────────────────────────────────────────
SCHMITT BUFFER ALLOCATION SUMMARY (U_SCH – SN54SC6T17-SEP)
────────────────────────────────────────────────────────────────────────

U_SCH has 6 Schmitt buffer gates. Usage:

  Gate #1: TIMING_A → PGOOD_DELAYED (startup delay timer)
  Gate #2: TIMING_B → RETRY_ACTIVE (retry timer)
  Gates #3-6: UNUSED (tie all inputs to GND)

8. SYSTEM INTEGRATION AND SIGNAL FLOW
================================================================================

COMPLETE FAULT DETECTION AND RECOVERY SEQUENCE:

────────────────────────────────────────────────────────────────────────
PHASE 1: NORMAL OPERATION
────────────────────────────────────────────────────────────────────────

Power state:
  - 5V_IN present from spacecraft
  - P-FET ON (ENABLE_GLOBAL = HIGH)
  - Buck #1 running → 3.3V available
  - All 6 rails operating normally

Signal states:
  1. Current draw normal on all rails (≤100% rated)
  2. V_AMP_1 through V_AMP_6: all <0.996V (e.g., ~660mV at 100% load)
  3. Comparators: V_AMP (e.g. 660mV) < V_ref (996mV) → IN+ < IN- → FAULT_n = LOW (all 6)
  4. OR cascade: all LOW → GLOBAL_FAULT = LOW
  5. SR latch: SET input = GLOBAL_FAULT = LOW → not triggered
  6. FAULT_LATCHED = LOW
  7. Q_dis: base driven HIGH (via inverter from FAULT_LATCHED = LOW)
  8. Q_dis ON → TIMING_B = 0V → RETRY_ACTIVE = LOW
  9. Startup delay: cap fully charged → PGOOD_DELAYED = HIGH
  10. Enable: PGOOD_DELAYED AND NOT(FAULT_LATCHED) = HIGH AND HIGH = HIGH
  11. Gate driver: ENABLE_GLOBAL = HIGH → GATE_DRIVE = LOW → P-FET ON ✓

────────────────────────────────────────────────────────────────────────
PHASE 2: LATCHUP EVENT
────────────────────────────────────────────────────────────────────────

T = 0µs: Cosmic ray strikes CPU, triggering latchup on 1.2V rail (Rail 3)

T = 0-5µs: Latchup current builds up
  - Rail 3 current: 2A → 5A → 10A
  - V_sense_3 = 10A × 10mΩ = 100mV
  - V_AMP_3 = 21.9 × 100mV = 2.19V

T = 5µs: Comparator detects fault
  - V_AMP_3 (2.19V) > V_ref (0.996V)
  - Comparator 3: IN+ > IN- → OUT = Hi-Z
  - FAULT_3 pulled HIGH by 10kΩ resistor to 5V

T = 5.01µs: Logic propagation
  - OR cascade: FAULT_3 = HIGH → GLOBAL_FAULT = HIGH
  - SR latch: SET input = GLOBAL_FAULT = HIGH → latch triggered
  - SR latch: FAULT_LATCHED goes HIGH
  - Inverter: FAULT_LATCHED = HIGH → Q_dis base = LOW
  - Q_dis turns OFF → TIMING_B can now charge
  - Inverter: FAULT_LATCHED = HIGH → NOT_FAULT_LATCHED = LOW
  - AND gate: PGOOD_DELAYED AND LOW = LOW
  - ENABLE_GLOBAL = LOW

T = 5.02µs: P-FET shutdown begins
  - Gate driver: ENABLE_GLOBAL = LOW → GATE_DRIVE = HIGH
  - P-FET gate pulled toward Source → Vgs rises toward 0V
  - Drain current begins to decrease

T = 6µs: P-FET fully off
  - Vgs = 0V → FET in cutoff
  - 5V_OUT collapses

T = 6-10ms: Power rails collapse
  - Buck #1 input gone → 3.3V output drops to 0V
  - All 6 downstream rails (3.3V, 1.2V, 1.8V, 2.5V, etc.) → 0V
  - Latchup current ceases (no power to sustain it)
  - Load components enter safe unpowered state

T = 10ms: Sensing domain dies
  - Op-amps (U_AMP_A, U_AMP_B) lose 3.3V power
  - V_AMP_n outputs → 0V or high-Z
  - Comparators see V_AMP = 0V < V_ref = 0.996V
  - Comparators output: IN+ < IN- → OUT = LOW → FAULT_n = LOW (false "no fault")
  - OR cascade: all LOW → GLOBAL_FAULT = LOW
  
  **CRITICAL:** Without SR latch, this would make system think no fault exists!
  
  **With SR latch:**
  - GLOBAL_FAULT = LOW → SET input de-asserted (was HIGH, now LOW)
  - But latch HOLDS its state: FAULT_LATCHED = HIGH (still remembering fault)
  - Q_dis still OFF → timer still charging ✓
  - ENABLE still LOW ✓
  - System correctly stays off ✓

────────────────────────────────────────────────────────────────────────
PHASE 3: RETRY TIMER RUNNING
────────────────────────────────────────────────────────────────────────

T = 10ms - 1.5s: Waiting period
  - Control domain (5V powered) stays alive
  - FAULT_LATCHED = HIGH (latch holding fault memory)
  - Q_dis OFF → TIMING_B capacitor charging: 0V → VT+ (1.43-2.71V)
  - Charge time: 0.7-1.7 seconds
  - ENABLE_GLOBAL = LOW (system stays off)
  - P-FET OFF
  - All rails remain at 0V
  - Components cool down, latchup fully extinguished

T = 1.2s (example, typ): Timer threshold reached
  - TIMING_B crosses VT+ (typ 2.0V)
  - Schmitt buffer: input > VT+ → RETRY_ACTIVE = HIGH

────────────────────────────────────────────────────────────────────────
PHASE 4: RESTART ATTEMPT
────────────────────────────────────────────────────────────────────────

T = 1.2s: RETRY_ACTIVE rising edge
  - RC differentiator: rising edge → RETRY_PULSE positive spike (~5ms wide)
  - Inverter: RETRY_PULSE = HIGH → RESET_L = LOW
  - SR latch: RESET triggered → FAULT_LATCHED = LOW
  - Q_dis base: inverted FAULT_LATCHED = LOW → base = HIGH
  - Q_dis turns ON → TIMING_B discharged to 0V
  - RETRY_ACTIVE returns LOW (cap discharged below VT-)

T = 1.201s: Enable logic updates
  - FAULT_LATCHED = LOW → NOT_FAULT_LATCHED = HIGH
  - PGOOD_DELAYED = HIGH (stayed high throughout)
  - AND gate: HIGH AND HIGH = HIGH
  - ENABLE_GLOBAL = HIGH

T = 1.202s: P-FET turns on
  - Gate driver: ENABLE_GLOBAL = HIGH → GATE_DRIVE = LOW
  - P-FET gate pulled to 0V → Vgs = -5V → FET ON
  - 5V_OUT appears

T = 1.203s: Buck converters start
  - Buck #1 sees 5V input → begins startup
  - Buck #2 sees 5V input → begins startup
  - Output capacitors begin charging
  
T = 1.203-1.400s: Startup delay active
  - TIMING_A capacitor charging (was discharged when power died)
  - PGOOD_DELAYED = LOW during this time
  - Enable: PGOOD_DELAYED = LOW → ENABLE_GLOBAL = LOW
  - Wait, this would turn OFF the FET again!
  
  **DESIGN ERROR CAUGHT:** TIMING_A capacitor was on Buck #1 3.3V in original
  design, so it died when Buck #1 died. Now we're on 5V, it stays charged!
  
  **FIX:** TIMING_A needs a discharge path when P-FET turns off.
  
  Add discharge transistor for startup delay:
    Q_start: small NPN (MMBT2222A)
      Collector → TIMING_A
      Emitter   → GND
      Base      → inverted ENABLE_GLOBAL (via 10kΩ)
  
  When ENABLE_GLOBAL = LOW → Q_start ON → TIMING_A discharged
  When ENABLE_GLOBAL = HIGH → Q_start OFF → TIMING_A can charge
  
  This ensures startup delay runs on every power-up attempt.

T = 1.400s (corrected): Startup delay completes
  - TIMING_A crosses VT+ → PGOOD_DELAYED = HIGH
  - Enable: PGOOD_DELAYED = HIGH, NOT_FAULT_LATCHED = HIGH
  - ENABLE_GLOBAL = HIGH (stays high)
  - P-FET stays ON

T = 1.450s: Buck outputs stabilize
  - Buck #1: 3.3V output reaches regulation
  - Op-amps power up
  - Buck #2 + PMIC: all 6 rails reach regulation
  - Load components begin normal operation

T = 1.500s: Sensing active
  - V_AMP_n outputs tracking actual rail currents
  - Comparators comparing against V_ref

**IF FAULT CLEARED (latchup was transient):**
  - All rails draw normal current
  - V_AMP_n < V_ref → IN+ < IN- → FAULT_n = LOW (all channels)
  - GLOBAL_FAULT = LOW
  - SR latch not triggered (SET input = LOW)
  - FAULT_LATCHED stays LOW (was already reset by timer)
  - Q_dis ON → timer cap stays discharged
  - System runs normally ✓✓✓

**IF FAULT PERSISTS (component permanently damaged):**
  - Rail 3 immediately latches again
  - V_AMP_3 > V_ref → IN+ > IN- → FAULT_3 = HIGH
  - GLOBAL_FAULT = HIGH
  - SR latch SET → FAULT_LATCHED = HIGH
  - ENABLE_GLOBAL = LOW
  - P-FET OFF
  - Cycle repeats (infinite autonomous retry)

────────────────────────────────────────────────────────────────────────
PHASE 5: INFINITE RETRY LOOP (if fault persists)
────────────────────────────────────────────────────────────────────────

The system will cycle:
  OFF 0.7-1.7s → ON attempt → fault re-detected in <10µs → OFF again

This continues indefinitely as long as 5V_IN is present.

OBC (spacecraft on-board computer) should monitor device telemetry:
  - If no telemetry for >10 seconds → stuck in retry loop
  - OBC can choose to:
    a) Remove 5V_IN completely (give up on this device)
    b) Switch to redundant backup device
    c) Wait for ground command

The hardware protection ensures:
  - No thermal damage to components (fast shutdown)
  - No permanent latch condition in protection circuit itself
  - Automatic recovery if fault clears (e.g., soft error, not hard failure)

10. PCB LAYOUT GUIDELINES
================================================================================

BOARD SIZE TARGET: 70mm × 50mm (reduced from 120mm × 80mm in v3.1)

LAYER STACKUP (4-layer minimum, 6-layer recommended):
  Layer 1: Top signal + high-current traces
  Layer 2: Ground plane (solid pour, no splits)
  Layer 3: 3.3V and 5V power planes (split)
  Layer 4: Bottom signal

CRITICAL LAYOUT RULES:

1. POWER DOMAIN SEPARATION:
   - 5V plane: covers entire board (control logic area)
   - 3.3V plane: only under op-amp area (sensing domain)
   - DO NOT connect 3.3V plane to Buck #1 output initially
   - Use single via connection point (star ground concept)
   - Physically separate 5V and 3.3V areas if possible

2. HIGH-CURRENT TRACES (Rails 1-2: 5A each, Shunts):
   - Trace width: 3mm minimum (2oz copper) for 3.3V AI core rails
   - Use top layer only (minimal vias in current path)
   - Shunt resistor pads: 4-wire Kelvin connection
     * Two pads for current (wide, 3mm traces)
     * Two pads for voltage sense (thin, 0.2mm traces to op-amp)
   - Keep sense traces <20mm long, parallel routing ±0.5mm
   - Ground plane cutout under shunt (avoid return current coupling)

3. DIFFERENTIAL PAIR ROUTING (V+ and V- to each op-amp):
   - Match lengths within 2mm
   - Parallel routing, 0.3mm spacing
   - Avoid crossing noisy signals (clock, digital logic)
   - Shield with ground via fence if crossing digital area

4. ANALOG-DIGITAL ISOLATION:
   - Op-amp area: top-left quadrant (quiet analog zone)
   - Logic area: bottom-right quadrant (noisy digital zone)
   - Ground: single point star connection at Q1 source pad
   - No digital return currents through analog ground area

5. COMPARATOR REFERENCE (V_ref):
   - 5 separate divider circuits (do not share single node)
   - Each divider has dedicated 100nF cap to GND
   - Place cap <2mm from comparator IN+ pin
   - Route V_ref traces away from switching nodes (P-FET gate, Buck outputs)

6. TIMING CAPACITORS:
   - C_start (330nF) and C_timer (2.2µF): low-ESR X7R ceramic
   - Place within 10mm of Schmitt buffer input pins
   - Guard ring around timing caps (GND via fence, 5mm spacing)
   - No switching traces near timing nodes

7. P-FET GATE DRIVER:
   - R1 (10kΩ pull-up): <5mm from gate pad
   - R2 (100Ω series) + C2 (100nF snubber): <3mm from gate pad
   - Gate trace: keep short (<15mm), no vias, 0.5mm width
   - Source connection: wide pour, multiple vias to 5V plane

8. CFP-14 PACKAGE (LMP7704-SP):
   - Hermetic flatpack, not reflow-compatible
   - Requires hand soldering or socket
   - Land pattern: consult LMP7704-SP datasheet Figure 42
   - Thermal relief: connect pins 1-14 to ground plane via thermal vias
   - Clearance around package: 3mm minimum (no components)

9. THERMAL MANAGEMENT:
   - Q1 (DPAK): large copper pour on drain pad, 10+ thermal vias to inner layers
   - Shunt resistors: thermal relief pads, expose to airflow
   - No components under Q1 or shunts (allow natural convection)

10. RADIATION SHIELDING CONSIDERATION:
    - PCB itself provides ~1mm Al-equivalent shielding
    - Additional 2mm Al sheet recommended over entire board (chassis mount)
    - Total shielding: ~3mm Al-eq → reduces TID to <10 krad over 3 years

11. TEST POINTS:
    - 5V_IN, 5V_OUT, 3.3V (Buck #1)
    - All 6 V_AMP_n signals (for trip threshold verification)
    - FAULT_1 through FAULT_6 (for fault injection testing)
    - GLOBAL_FAULT, FAULT_LATCHED, ENABLE_GLOBAL, GATE_DRIVE
    - TIMING_A, TIMING_B (for timer verification)
    - Place test points on 0.1" grid for easy probing

12. SILKSCREEN MARKINGS:
    - Power domains: "5V DOMAIN" and "3.3V DOMAIN" text
    - Signal names at all test points
    - Component reference designators
    - Polarity marks for Q1, Q_dis, Q_start
    - "LATCHUP PROTECTION v3.3" and date code

12. TESTING PROCEDURES
================================================================================

EQUIPMENT REQUIRED:
  - Bench power supply: 5V, 6A capability, current limiting
  - Electronic loads: 6× programmable DC loads (0-10A, 0-5V) for rail simulation
  - Oscilloscope: 4-channel, ≥100 MHz bandwidth, 1 GSa/s
  - DMM: 6.5 digit, current and voltage measurement
  - Load resistors: assorted (1Ω to 10Ω, 10W+ rated) for fixed load testing
  - MOSFET switches: 6× N-channel FETs (IRF540 or similar) for fault injection
  - Function generator: for pulse injection (optional, advanced testing)

────────────────────────────────────────────────────────────────────────
TEST 1: POWER-ON AND QUIESCENT CURRENT
────────────────────────────────────────────────────────────────────────

Setup:
  - No loads connected to any rail
  - Buck converters and PMIC present but no downstream loads
  - Set power supply to 5V, current limit 100mA

Procedure:
  1a. Apply 5V to 5V_IN
  1b. Observe 5V_OUT with oscilloscope (trigger on rising edge)
  1c. Measure time from 5V_IN rise to 5V_OUT rise: should be 110-260ms
  1d. Verify 5V_OUT reaches 5V (within 50mV) after delay
  1e. Measure current draw from 5V_IN: should be <10mA

Pass criteria:
  - Startup delay: 80-300ms (allows temp variation margin)
  - 5V_OUT steady state: 4.95V - 5.05V
  - Quiescent current: <15mA

Fail diagnosis:
  - No 5V_OUT: Check Q1 orientation, ENABLE_GLOBAL stuck low, gate driver
  - Immediate 5V_OUT (no delay): C_start open, U_SCH damaged, short on TIMING_A
  - Excessive current: Short circuit, check for solder bridges

────────────────────────────────────────────────────────────────────────
TEST 2: RAIL-BY-RAIL CURRENT SENSE CALIBRATION
────────────────────────────────────────────────────────────────────────

Purpose: Verify each channel detects overcurrent at correct threshold.

Setup:
  - Load each rail individually with programmable DC electronic load
  - Start with Rail 1 (3.3V, trip at 7.5A)
  - Oscilloscope channels:
    * Ch1: V_AMP_1 (op-amp output)
    * Ch2: FAULT_1 (comparator output)
    * Ch3: 5V_OUT (to see shutdown)
    * Ch4: ENABLE_GLOBAL (to see enable control)

Procedure for Rail 1:
  2a. Set electronic load to constant current mode, 0A
  2b. Verify Buck #1 is outputting 3.3V on Rail 1
  2c. Gradually increase load current: 0A → 1A → 2A → 3A → 4A → 5A
  2d. At each step, measure V_AMP_1 voltage:
      * 1A: ~132mV (13.18 × 10mΩ × 1A)
      * 2A: ~264mV
      * 3A: ~395mV
      * 4A: ~527mV
      * 5A: ~659mV (100% rated load, still below 996mV trip point)
  2e. Continue increasing: 5.5A → 6A → 6.5A → 7A → 7.5A
  2f. Observe V_AMP_1 crossing 996mV threshold
  2g. Expected trip point: 7.3A - 7.7A (allows for tolerances)
  2h. When V_AMP_1 > 996mV:
      * FAULT_1 should go HIGH (comparator releases, pull-up pulls high)
      * 5V_OUT should collapse within 10µs
      * System shuts off
  2i. Wait 1-2 seconds, observe automatic restart attempt
  2j. Reduce load to 5A before restart completes
  2k. Verify system recovers and runs normally (FAULT_1 returns to LOW)

Repeat for Rails 2-6 with appropriate trip currents:
  - Rail 2: Trip at 7.3-7.7A (same as Rail 1)
  - Rail 3: Trip at 4.3-4.7A
  - Rail 4: Trip at 0.57-0.63A (100mΩ shunt, more sensitive)
  - Rail 5: Trip at 2.85-3.15A
  - Rail 6: Trip at 1.43-1.57A

Pass criteria:
  - V_AMP_n within 200mV of calculated value at 100% load
  - Trip occurs within ±5% of designed threshold (e.g., 7.5A ±0.375A)
  - Shutdown time <15µs from trip to 5V_OUT collapse
  - Automatic restart after 0.5-2.0 seconds
  - Clean recovery when load reduced

Fail diagnosis:
  - V_AMP too high: Gain resistor wrong value, check R2/R4
  - V_AMP too low: Shunt resistor wrong value, or poor Kelvin connection
  - No trip: Comparator inputs swapped, V_ref dead, comparator damaged
  - No shutdown: Logic chain broken, check ENABLE_GLOBAL path
  - No restart: SR latch stuck, timer not working, check FAULT_LATCHED

────────────────────────────────────────────────────────────────────────
TEST 3: RESPONSE TIME MEASUREMENT
────────────────────────────────────────────────────────────────────────

Purpose: Verify protection responds fast enough to prevent thermal damage.

Setup:
  - Use MOSFET switch to create step-change overcurrent fault
  - Rail 3 (slowest expected, but Rail 6 is actually slowest)
  - Test with Rail 6 (1.2V, trip at 1.5A) for worst-case verification

Circuit:
    Buck output (1.2V) ──[R_load: 0.82Ω, 10W]── MOSFET drain
                                                MOSFET source ─┬─ GND
                                                               │
    Shunt measurement point                                    │
                                                               │
    Function generator ───────────────────────── MOSFET gate ─┘
    (0V → 5V pulse, 10µs rise time)

When MOSFET turns ON: load current jumps from 0A to 1.2V/0.82Ω ≈ 1.46A (just below trip)
Increase to 0.68Ω for 1.76A fault (above trip threshold)

Oscilloscope setup (10µs/div, AC couple, 10× probes):
  Ch1: V_sense across shunt (50mV/div)
  Ch2: V_AMP_6 (500mV/div)
  Ch3: FAULT_6 (2V/div)
  Ch4: 5V_OUT (2V/div)

Procedure:
  3a. Set function generator to output single pulse: 0V → 5V, 10µs wide
  3b. Trigger oscilloscope on Ch1 (V_sense rising edge)
  3c. Apply pulse (MOSFET conducts, fault current flows)
  3d. Measure time intervals:
      * T1: V_sense 10% → 90% (current rise)
      * T2: V_sense 50% → V_AMP_6 crossing 996mV (amplifier delay)
      * T3: V_AMP_6 crossing 996mV → FAULT_6 falling edge (comparator delay)
      * T4: FAULT_6 falling edge → 5V_OUT 90% → 10% (logic + FET delay)
      * T_total: V_sense 50% → 5V_OUT 10% (total response time)

Expected results (Rail 6, worst case):
  - T2 (amplifier): 8-10µs
  - T3 (comparator): 0.5-1.5µs
  - T4 (logic + FET): 0.3-0.5µs
  - T_total: 9-12µs

Pass criteria:
  - T_total <15µs for all rails
  - Rail 1-2 (fastest): <5µs
  - Rail 6 (slowest): <12µs

Fail diagnosis:
  - Slow amplifier (>15µs): Check power supply, verify 3.3V on op-amps
  - Slow comparator (>2µs): Check V_ref, verify 5V supply
  - Slow shutdown (>1µs): Check gate driver, C2 too large, R2 too large

────────────────────────────────────────────────────────────────────────
TEST 4: INFINITE RETRY VERIFICATION
────────────────────────────────────────────────────────────────────────

Purpose: Verify system retries indefinitely with correct delay timing.

Setup:
  - Sustained fault on Rail 3 (permanent short via resistor)
  - Load: R = 0.27Ω, 10W (creates 1.2V/0.27Ω = 4.4A > 4.5A trip)

Oscilloscope (1 second/div, long record length 10+ seconds):
  Ch1: 5V_OUT (2V/div)
  Ch2: FAULT_LATCHED (2V/div)
  Ch3: TIMING_B (1V/div)
  Ch4: RETRY_ACTIVE (2V/div)

Procedure:
  4a. Power on system with fault load connected
  4b. Observe first shutdown (immediate on power-up after startup delay)
  4c. Measure time from 5V_OUT falling to next 5V_OUT rising (retry period)
  4d. Expected: 0.7s - 1.7s
  4e. Let system cycle 10 times, record each retry period
  4f. Verify all periods within 0.5s - 2.0s range (allows variation)
  4g. Confirm no permanent latch (system never gives up)

Expected waveforms:
  - 5V_OUT: pulses high for ~200ms (startup delay), then drops
  - FAULT_LATCHED: stays HIGH during off period
  - TIMING_B: ramps up 0V → 2V over 0.7-1.7s, then resets
  - RETRY_ACTIVE: brief HIGH pulse when TIMING_B crosses threshold

Pass criteria:
  - Minimum 10 consecutive retry cycles observed
  - Retry delay: 0.5s - 2.0s per attempt
  - No permanent shutdown state

Fail diagnosis:
  - No retry: SR latch not resetting, check RESET_L signal
  - Timer doesn't charge: Q_dis stuck ON, check inverter gate #4
  - Immediate retry (<0.1s): C_timer wrong value or shorted
  - Long retry (>3s): R_timer wrong value or Schmitt threshold issue

────────────────────────────────────────────────────────────────────────
TEST 5: STARTUP DELAY VERIFICATION
────────────────────────────────────────────────────────────────────────

Purpose: Verify inrush current during power-up doesn't cause false trip.

Setup:
  - All 6 rails connected to realistic loads (actual CPU, AI cores, memory)
  - Worst-case inrush: capacitive loads (bulk caps on rail outputs)

Oscilloscope (50ms/div):
  Ch1: 5V_IN (trigger source)
  Ch2: 5V_OUT
  Ch3: 3.3V (Buck #1 output)
  Ch4: PGOOD_DELAYED

Procedure:
  5a. Remove power completely, wait 10 seconds
  5b. Apply 5V to 5V_IN
  5c. Observe PGOOD_DELAYED delay before going HIGH
  5d. Verify 5V_OUT appears only after PGOOD_DELAYED = HIGH
  5e. Measure startup delay: 5V_IN rising edge to 5V_OUT stable
  5f. Repeat 5 times, record values

Pass criteria:
  - Startup delay: 80-300ms (allows for threshold variation)
  - No false trips during inrush (all FAULT_n stay HIGH)
  - All rails reach regulation before protection active

Fail diagnosis:
  - False trip during startup: Startup delay too short, increase C_start
  - No power output: Q_start not discharging TIMING_A, check circuit
  - Erratic delay: C_start leaky, replace

────────────────────────────────────────────────────────────────────────
TEST 6: CROSS-TALK AND NOISE IMMUNITY
────────────────────────────────────────────────────────────────────────

Purpose: Verify one rail's activity doesn't false-trigger another rail.

Setup:
  - Load Rail 1 with pulsed load (50% duty cycle, 3A → 6A square wave, 1kHz)
  - Monitor all other FAULT_n signals
  - Verify no crosstalk-induced trips

Oscilloscope:
  Ch1: Rail 1 current (via shunt voltage)
  Ch2-4: FAULT_2, FAULT_3, FAULT_4 (or use logic analyzer for all 6)

Procedure:
  6a. Apply pulsed load to Rail 1 for 60 seconds
  6b. Verify all other FAULT_n signals stay LOW (no glitches >1µs)
  6c. Repeat for each rail individually

Pass criteria:
  - No false triggers on unloaded rails (FAULT_n stay LOW)
  - FAULT_n signals clean (no glitches >100ns)

Fail diagnosis:
  - Crosstalk: Poor PCB layout, sense traces too close to power traces
  - Ground bounce: Inadequate ground plane, add more vias

────────────────────────────────────────────────────────────────────────
TEST 7: THERMAL CYCLING (OPTIONAL, FOR FLIGHT QUALIFICATION)
────────────────────────────────────────────────────────────────────────

Purpose: Verify design operates over LEO temperature range.

Temperature range: -40°C to +85°C (typical LEO orbit thermal extremes)

Procedure:
  7a. Place assembled board in thermal chamber
  7b. Run Test 2 (rail-by-rail calibration) at -40°C
  7c. Repeat at +25°C (room temp baseline)
  7d. Repeat at +85°C
  7e. Compare trip thresholds at all temperatures

13. FAILURE MODE ANALYSIS
================================================================================

FAILURE 1: False trip during normal operation
  Cause: Noise on sense traces, transient inrush, op-amp oscillation
  Detection: System shuts off briefly, restarts after ~1 second
  Mitigation:
    - 150% threshold provides margin vs. steady-state noise
    - Input filter caps on op-amp inputs (100nF) reduce HF noise
    - Startup delay prevents false trips during power-on
  Likelihood: Low (design margins adequate)
  Impact: Minor (automatic recovery within 1-2 seconds)

FAILURE 2: Missed latchup (no detection)
  Cause: Response time too slow, OR latchup current below 150% threshold
  Likelihood: Low on Rails 1-5 (response <6µs), moderate on Rail 6 (9µs)
  Analysis:
    - Macro-SEL develops over 10-100µs timescale
    - Even Rail 6 (9µs) has margin vs. typical 50µs threat
    - If latchup current stays <150% (e.g., micro-SEL), it's self-quenching
  Impact: If missed, component may suffer thermal damage
  Mitigation: 150% threshold is conservative; most latchups far exceed this
  Residual risk: Accepted for hardware-only design

FAILURE 3: Protection circuit itself latches (CMOS logic SEL)
  Cause: Single-event latchup in TLV1704-SEP or SN54SC logic IC
  SEL LET thresholds: 43 MeV·cm²/mg (comparators and logic)
  LEO particle spectrum: Most below this threshold, but HZE ions can exceed
  Likelihood: Low (<1 event per year estimated)
  Impact:
    - If comparator latches: false FAULT signal, unnecessary shutdown
    - If SR latch IC latches: stuck in fault state, no restart
    - If gate driver latches: P-FET stuck ON or OFF
  Detection: OBC monitors device telemetry
    - If device cycling without recovery (>5 retry cycles): protection stuck
    - If device unresponsive for >30 seconds: protection or load failed
  Mitigation: OBC should power-cycle 5V_IN to clear protection circuit latchup
  Note: Op-amps (LMP7704-SP) have LET = 85 MeV·cm²/mg, very robust

FAILURE 4: TID degradation over mission life
  Components at risk:
    - SN54SC logic and TLV1704-SEP: 30 krad rated
    - LMP7704-SP op-amps: 100 krad rated (not at risk)
  Expected dose: 9-45 krad over 3 years (depends on shielding, solar activity)
  Effects at 30 krad:
    - Logic gate threshold shifts <100mV (within noise margin)
    - Comparator offset increases to ~5mV (negligible vs. 996mV threshold)
    - MOSFET R_DS(on) increase <10% (acceptable)
  Mitigation: Minimum 2mm Al shielding recommended to stay <15 krad
  Likelihood of failure: Very low if shielded properly

FAILURE 5: Stuck in infinite retry loop (permanent component damage)
  Cause: COTS component permanently damaged, latchup repeats on every restart
  Detection: OBC monitors device status
    - No telemetry for >10 seconds → likely stuck in retry
    - Power consumption pattern: ON 200ms, OFF 1s, repeating
  OBC response:
    - After 10 failed retry cycles (~15 seconds), remove 5V_IN
    - Log fault for ground analysis
    - Switch to backup/redundant device if available
  Impact: Device offline until ground intervention
  This is EXPECTED behavior for permanent failures (protection working correctly)

FAILURE 6: Q1 thermal failure
  Cause: Sustained high current before shutdown, or repeated latchup stress
  Analysis:
    - Steady-state: 1.1W dissipation, 55°C rise, safe
    - Fault pulse: 50A × 10µs = 500µA·s = 2.5mJ energy
    - Avalanche rating: 500mJ (200× margin)
    - Repeated cycling: Each retry = 2.5mJ pulse, 1000 cycles = 2.5J total
  Likelihood: Extremely low
  Mitigation: Q1 specified for 10,000+ switching cycles

FAILURE 7: Buck #1 failure (sensing domain loses power permanently)
  Cause: Buck #1 fails due to radiation or component wear
  Impact:
    - Op-amps lose power → no current sensing
    - Comparators see V_AMP = 0V → false "no fault" signal
    - SR latch not triggered (GLOBAL_FAULT stays HIGH)
    - System appears healthy but has no protection
  Detection: Difficult to detect without external monitoring
  Mitigation:
    - Monitor Buck #1 3.3V output via telemetry
    - If 3.3V absent but 5V_OUT present → Buck #1 failed, alarm
  Likelihood: Low (Buck converter is commercial part, but high reliability)
  Recovery: OBC should shut down device if Buck #1 fails

FAILURE 8: SR latch fails to set (stuck in reset state)
  Cause: NAND gate damaged, GLOBAL_FAULT not inverting properly
  Impact: Fault detected but not remembered, system shuts down briefly then
          restarts immediately (no retry delay)
  Detection: Rapid power cycling (ON/OFF at ~10Hz instead of 1Hz retry)
  Mitigation: If detected, OBC removes power to prevent thermal stress
  Likelihood: Low (NAND gates are simple, robust)

14. OPEN ITEMS AND LIMITATIONS
================================================================================

OPEN ITEM 1: CFP-14 Assembly Process (BLOCKING for PCB finalization)
  The LMP7704-SP CFP-14 hermetic flatpack is not reflow-compatible.
  Options:
    a) Hand-solder all 14 pins (requires skilled technician, ~30 min per IC)
    b) Use CFP-14 socket (Samtec or Mill-Max) – adds mechanical risk
    c) Request LMP7704-SP in different package (not available)
  Decision needed before PCB layout finalized.
  Recommendation: Hand-solder with documented process control (temperature,
                   time per pin, visual inspection requirements)

OPEN ITEM 2: Q_start Discharge Transistor (NEW in v3.3)
  Added Q_start (MMBT2222A) to discharge TIMING_A when P-FET is off.
  This ensures startup delay runs on every power-up attempt, not just first.
  Circuit needs verification on breadboard before PCB layout.
  Verify Q_start doesn't introduce oscillation or race conditions.

OPEN ITEM 3: RETRY_PULSE Generation (NEW in v3.3)
  RC differentiator (10nF + 100kΩ) generates reset pulse for SR latch.
  Pulse width ~5ms may be too long (wastes power keeping RETRY_ACTIVE high).
  Alternative: Use one-shot circuit or decrease RC values.
  Test on breadboard to confirm latch reset works reliably.

OPEN ITEM 4: Confirm ARM Cortex-A72 Process Node
  CPU and AI core process nodes unknown (confidential from user).
  Assumed 28nm planar (worst case for latchup susceptibility).
  If actual process is 14nm/16nm FinFET:
    - Latchup threshold higher (less susceptible)
    - 150% trip threshold may be too conservative (could miss weak latchups)
  Recommendation: Request process node info from CPU/AI vendor, or perform
                   heavy-ion latchup testing to empirically determine threshold

OPEN ITEM 5: Rail 4 Voltage Drop Budget (CRITICAL)
  100mΩ shunt on 1.35V rail causes 40mV drop at 400mA rated current.
  CPU core voltage tolerance typically ±5% = ±67.5mV.
  40mV leaves only 27.5mV margin for PCB trace + connector drop.
  After PCB layout: measure total drop from PMIC output to CPU input.
  If drop >67.5mV at 400mA, must reduce shunt to 75mΩ or 50mΩ.
  Trade-off: Lower shunt = less trip precision (gain increases, noise margin reduces)

OPEN ITEM 6: V_ref Temperature Coefficient
  V_ref divider (40.2kΩ / 10kΩ) uses 0.1% resistors.
  Tempco not specified (E96 series typically ±50 ppm/°C).
  Over -40°C to +85°C (125°C span): V_ref drift = ±6.25mV worst case.
  This is 0.6% of 996mV reference, acceptable.
  If tighter tolerance needed (±0.2%), use thin-film resistors (±25 ppm/°C).

OPEN ITEM 7: Buck #1 Startup Current (inrush)
  When P-FET turns on, Buck #1 input capacitors charge.
  If C_in = 100µF (typical), charge current = C × dV/dt.
  At 1µs turn-on: I_inrush = 100µF × 5V / 1µs = 500A peak!
  This will trip overcurrent protection on Rails 1-2 (fed from Buck #1).
  Mitigation:
    - Add soft-start to Buck #1 (internal feature, check datasheet)
    - OR increase startup delay to 500ms (gives Buck #1 time to soft-start)
  Decision: Increase C_start to 680nF (gives 220-520ms startup delay)

LIMITATION 1: No per-rail isolation
  All rails cut simultaneously when any one faults.
  A latchup on Rail 4 (400mA CPU rail) shuts down Rail 1 (5A AI core).
  This is intentional – individual FETs per rail would require 6× BUP06CP
  parts, massive board area increase.
  OBC can monitor which FAULT_n triggered (via telemetry) for diagnostics.

LIMITATION 2: Micro-SEL not detected
  Events that resolve in <10µs will not trip protection.
  Acceptable: Micro-SEL is self-quenching in modern CMOS, no permanent damage.

LIMITATION 3: Retry delay varies 2.4:1 (0.7-1.7s range at 5V)
  Schmitt trigger threshold tolerance is ±40% (min 1.43V, max 2.71V at 5V).
  This causes wide timing variation (factor of 2.4×).
  Not a problem functionally – any delay >0.5s is adequate for latchup recovery.
  If precise timing needed, use precision timer IC (LTC6995 or similar), but
  adds component count and radiation qualification burden.

LIMITATION 4: Protection circuit single point of failure
  If SR latch, comparators, or gate driver fail, entire protection is lost.
  No redundancy in protection circuit (cost/size/complexity trade-off).
  Mitigation: OBC-level monitoring provides backup (can detect and remove power).
  For critical missions: Consider dual protection circuits with voting logic.

LIMITATION 5: 5V_IN must stay on for autonomous retry
  If OBC removes 5V_IN, system will not restart automatically.
  This is by design – OBC can disable device if stuck in fault loop.
  Ensure OBC power switching logic is documented and tested.
