1. SYSTEM ARCHITECTURE
================================================================================

PROTECTION STRATEGY:
  One master P-channel MOSFET (high-side, 5V input rail)
  Six independent current-sensing channels (one per downstream rail)
  Any fault on any rail trips all rails simultaneously
  Hardware-only infinite retry: ~1 second off between attempts
  OBC monitors device health and cuts input power if stuck in retry loop

RESPONSE TIME (worst case, Rail 6):
  Difference amplifier:  9.2 µs  (Rail 6, highest gain)
  Comparator:            0.6 µs  (TLV1704-SEP typical propagation delay)
  Logic chain:          <0.01 µs (CMOS propagation, negligible)
  P-FET turn-off:        0.3 µs  (gate RC = 100Ω × 2.5nF)
  TOTAL:                ~10.1 µs worst case

RESPONSE TIME (fastest, Rail 1 or 2):
  Difference amplifier:  1.8 µs
  Rest of chain:         0.9 µs
  TOTAL:                ~2.7 µs

ADEQUACY: Macro-SEL events develop over 10–1000+ µs before causing
permanent damage. Both figures are well within the safe response window
for macro-SEL. Micro-SEL (sub-µs pulse) is not detected, but micro-SEL
on advanced CMOS nodes is typically self-recovering.

TOTAL shunt power at rated currents: 656 mW
The shunt on Rail 1 and Rail 2 (250 mW each) require 1W-rated packages.

Trip threshold rationale: 150% catches latchup (which typically drives
5–20× rated current) while avoiding false trips from boot-time inrush
(which can be 3–5× rated, so 150% is above the inrush window but
substantially below the latchup window). The two do not overlap.

PROTECTION CIRCUIT POWER SUPPLY:
  Logic, comparators, op-amps, and V_ref dividers powered from Buck #1
  3.3V output (drawn before the shunt resistors on Rail 1).
  Current draw: approximately 20–30 mA total.
  When master FET turns off → Buck #1 loses input → 3.3V collapses
  → protection logic powers down → timer state resets naturally.
  This is intentional. Timer starts fresh on every retry attempt.

3. MASTER PROTECTION SWITCH
================================================================================

PURPOSE: Single P-channel MOSFET on 5V input rail. Turns OFF within 300 ns
of gate drive signal, cutting power to all downstream buck converters and PMIC
simultaneously. This is the recovery mechanism.

Q1 — BUP06CP038F-01
────────────────────────────────────────────────────────────────────────────────

  Manufacturer:   Infineon Technologies
  Part number:    BUP06CP038F-01
  Package:        DPAK (TO-252-3, standard SMD reflow compatible)
  Type:           P-channel power MOSFET

  KEY ELECTRICAL:
    V_DS:          -60 V
    I_D (25°C):    -35 A continuous
    I_D (100°C):   -28 A continuous
    I_DM (peak):   -140 A
    P_tot:         125 W at TC ≤ 25°C

    C_iss:         2.5 nF typical
    t_d(off):      47 ns typical

  RADIATION:
    TID:           30–50 krad(Si)
    SEE LET:       46 MeV·cm²/mg (designed for 2–5 year LEO missions)
    Note: DMOS power MOSFET structure — inherently more radiation-tolerant
          than CMOS due to different gate oxide physics

  THERMAL (at 5A nominal load):
    P_D = I² × R_DS = 5² × 0.044 = 1.1 W at rated current
    ΔT_junction = 1.1 W × R_th,JC = 1.1 × 1.2 = 1.3°C above case

  OPERATING POINT:
    Source: connected to +5V INPUT
    Gate:   driven by gate driver circuit below
    Drain:  output → 5V protected output → downstream converters
    V_GS when ON:  Gate = 0V, Source = +5V → V_GS = −5V

      the edge of enhancement — but actual threshold is typ −3V, so in
      practice V_GS = −5V drives well into the linear region.
      R_DS(on) ≈ 44 mΩ, acceptable for 5A operation.
    V_GS when OFF: Gate ≈ 3.3–5V → V_GS = −1.7V to 0V, below threshold → OFF

────────────────────────────────────────────────────────────────────────────────
GATE DRIVE CIRCUIT
────────────────────────────────────────────────────────────────────────────────

  Inverter:  One gate of SN54SC6T14-SEP (TSSOP-14, hex Schmitt inverter)
             Powered at 3.3V from Buck #1

  Rationale for P-channel vs N-channel:
    Source at +5V; N-channel would require gate >7V (above supply).
    P-channel naturally driven by pulling gate to GND — no charge pump needed.

  Schematic:

    +5V INPUT
        │
       [R1: 10 kΩ, 1206]    ← failsafe pull-up: if logic loses power,
        │                       gate defaults to +5V → V_GS = 0 → FET OFF
        ├──────────────────────────────────── GATE (Q1 pin 1)
        │                                          │
       [R2: 100 Ω, 1206]                    [C2: 100 nF, 1206 C0G]
        │                                          │
        │ INV_OUT                           +5V INPUT
        │
    [SN54SC6T14-SEP]  ← one inverter gate
        │ IN
        │
    ENABLE_GLOBAL (from logic section)

  Operation:
    ENABLE_GLOBAL = HIGH (3.3V) → INV_OUT = LOW (0V) → V_GS = 0V − 5V = −5V → FET ON
    ENABLE_GLOBAL = LOW (0V)   → INV_OUT = HIGH (3.3V) → V_GS ≈ 3.3V − 5V = −1.7V → FET OFF

  Gate switching time: R2 × C_iss = 100Ω × 2.5 nF = 250 ns → fast, controlled slew

  Full SN54SC6T14-SEP connections:
    Used gate: input = ENABLE_GLOBAL, output = INV_OUT → R2 → GATE
    Unused inputs (5 remaining): tie to GND via 10kΩ or directly to GND
    Unused outputs: leave unconnected
    Pin VCC: +3.3V (Buck #1)
    Pin GND: PCB ground plane
    Bypass cap: 100 nF ceramic (X7R) from VCC to GND, place within 2 mm of IC

────────────────────────────────────────────────────────────────────────────────
INPUT PROTECTION RESISTORS (ALL CHANNELS)
────────────────────────────────────────────────────────────────────────────────

  The LMP7704-SP has ±0.3V differential input absolute maximum. In closed-loop
  operation the virtual short prevents violation. However, during power-up or
  the brief moment after a latchup transient collapses the rail faster than the
  op-amp responds, the differential can exceed this limit.

  MITIGATION: R_p = 1.0 kΩ in series with EACH of the four input pins
  (both IN+ and IN−) for every channel. The ESD clamp diodes limit the
  differential via I = ΔV / R_p. Maximum case: I = 3.3 / 1.0k = 3.3 mA
  — well within the 10 mA absolute max current rating.

  GAIN COMPENSATION: R_p adds to R1 and R3 in the signal path. To maintain
  the design gain, R1 and R3 are chosen as 9.09 kΩ (not 10 kΩ), so that:
    R1_eff = R1 + R_p = 9.09 kΩ + 1.0 kΩ = 10.09 kΩ
  BOTH inputs receive R_p, so matching is preserved and differential gain
  remains determined by R2/R1.

  CRITICAL: R_p resistors (1.0 kΩ) for each channel must come from the
  same batch tape. Do not mix batches for the two R_p in the same channel.

5. CURRENT SENSE CIRCUITS — STAGE 2: COMPARATORS
================================================================================

PURPOSE: Compare each V_AMP_n against the reference threshold V_ref.
When V_AMP_n > V_ref → normal, FAULT_n = HIGH
When V_AMP_n ≤ V_ref → overcurrent, FAULT_n = LOW (triggers trip)

────────────────────────────────────────────────────────────────────────────────
TLV1704-SEP — COMPARATOR ALLOCATION
────────────────────────────────────────────────────────────────────────────────

  U_COMP_A: TLV1704-SEP (TSSOP-14)
    Comparators 1–4 → Rails 1, 2, 3, 4

  U_COMP_B: TLV1704-SEP (TSSOP-14)
    Comparators 1–2 → Rails 5, 6
    Comparators 3–4 → UNUSED: IN+ to GND, IN− to 3.3V (output held LOW,
                      tie pull-up resistor to 3.3V so output is HIGH = no fault)

  TLV1704-SEP key specs:
    Supply voltage:    2.2–24V (powered at 3.3V)
    Output type:       Open drain (requires pull-up resistor)
    Propagation delay: 560 ns typical
    Input offset:      ±300 µV
    Common-mode:       Rail-to-rail
    TID:               30 krad(Si) RLAT
    SEE LET:           43 MeV·cm²/mg

  TLV1704-SEP pinout (TSSOP-14):
    Comp 1: IN+(1) = pin 4, IN−(1) = pin 3, OUT(1) = pin 2
    Comp 2: IN+(2) = pin 6, IN−(2) = pin 7, OUT(2) = pin 8
    Comp 3: IN+(3) = pin 9, IN−(3) = pin 10, OUT(3) = pin 11
    Comp 4: IN+(4) = pin 13, IN−(4) = pin 12, OUT(4) = pin 1
    GND = pin 5,  VCC = pin 14

  ALL six comparators wired as:
    IN+(n): V_AMP_n  (from difference amplifier output)
    IN−(n): V_ref node (0.994V, see below)
    OUT(n): → R_pull (10 kΩ to 3.3V) → FAULT_n signal

  Output logic:
    V_AMP_n > V_ref → OUT = Hi-Z → R_pull pulls HIGH → FAULT_n = HIGH (no fault)
    V_AMP_n ≤ V_ref → OUT = LOW  → FAULT_n = LOW  (FAULT DETECTED)

  Pull-up resistors R_pull = 10 kΩ to 3.3V — one per comparator output (6 total)

────────────────────────────────────────────────────────────────────────────────
V_REF GENERATION
────────────────────────────────────────────────────────────────────────────────

  Target: 1.0V (matches V_AMP_n at trip point)
  Supply: 3.3V (from Buck #1)

  Formula: V_ref = R_bot / (R_top + R_bot) × 3.3V = 1.0V
  → R_top / R_bot = 2.3

  SELECTED (E96, 0.1%):
    R_top = 23.2 kΩ
    R_bot = 10.0 kΩ
    V_ref = 10.0 / (23.2 + 10.0) × 3.3 = 0.994V  (−0.6% error — acceptable)

  GROUPING: Rails 1 & 2 share one V_ref divider (identical trip voltage).
  Rails 3, 4, 5, 6 each have their own divider (same values).
  Total: 5 dividers × 2 resistors = 10 resistors.

  DECOUPLING: One 100 nF ceramic (X7R) capacitor from each V_ref node
  to GND. Prevents comparator switching transients from modulating V_ref.
  Place within 2 mm of each comparator IN− pin.

  V_ref TOLERANCE vs TRIP ACCURACY:
    V_AMP_n at trip ranges from 985 mV to 1005 mV (see channel table)
    V_ref = 994 mV
    Trip will occur when V_AMP_n exceeds 994 mV.
    For Rail 4 (V_AMP = 1005 mV at calculated trip): margin = 11 mV above V_ref ✓
    For Rail 3 (V_AMP = 985 mV at calculated trip): 985 < 994, meaning the
      comparator would trip at slightly less than 150% — approximately 147%.
      This is acceptable; the 150% figure is a target, not a hard guarantee.
      Any trip between 140% and 160% is functionally correct.

────────────────────────────────────────────────────────────────────────────────
HYSTERESIS (OPTIONAL)
────────────────────────────────────────────────────────────────────────────────

  The TLV1704-SEP is an open-drain comparator with no built-in hysteresis.
  A 1 MΩ positive feedback resistor from OUT to IN+ adds ~3 mV of hysteresis
  (prevents chatter at the threshold boundary). This is optional but recommended
  for noisy environments. If added: R_hyst = 1 MΩ from FAULT_n to V_AMP_n.

6. LOGIC CONTROLLER
================================================================================

All SN54SC logic ICs: powered at 3.3V from Buck #1. TID = 30 krad. TSSOP-14.

────────────────────────────────────────────────────────────────────────────────
SECTION A: FAULT AGGREGATION (6-input OR using cascade of 2-input gates)
────────────────────────────────────────────────────────────────────────────────

  IC: U_OR — SN54SC4T32-SEP (quad 2-input OR gate, TSSOP-14)
  Need: 2 chips (8 gates total; 5 used in 3-stage cascade)

  Signal names: F1 = FAULT_1, F2 = FAULT_2, ... F6 = FAULT_6 (all HIGH = OK)
  Logic: GLOBAL_FAULT = F1 OR F2 OR F3 OR F4 OR F5 OR F6

  Implementation (5 gates, 2 ICs):

    Stage 1 (IC U_OR_A):
      Gate 1: A = F1,  B = F2  → out = F12
      Gate 2: A = F3,  B = F4  → out = F34
      Gate 3: A = F5,  B = F6  → out = F56
      Gate 4: UNUSED — tie both inputs to GND

    Stage 2 (IC U_OR_B):
      Gate 1: A = F12, B = F34 → out = F1234
      Gate 2: A = F1234, B = F56 → out = GLOBAL_FAULT
      Gates 3, 4: UNUSED — tie both inputs to GND

  GLOBAL_FAULT = HIGH when all rails normal
  GLOBAL_FAULT = LOW when ANY rail in overcurrent

  SN54SC4T32-SEP pinout (TSSOP-14):
    Gate 1: A=pin 1, B=pin 2, Y=pin 3
    Gate 2: A=pin 4, B=pin 5, Y=pin 6
    GND=pin 7
    Gate 3: A=pin 9, B=pin 10, Y=pin 8
    Gate 4: A=pin 12, B=pin 13, Y=pin 11
    VCC=pin 14

  Power: 3.3V to pin 14, GND to pin 7.
  Bypass: 100 nF ceramic from pin 14 to pin 7, within 2 mm.

────────────────────────────────────────────────────────────────────────────────
SECTION B: STARTUP DELAY (~300 ms)
────────────────────────────────────────────────────────────────────────────────

  PURPOSE: Prevent false tripping during power-on inrush before rails settle.
  The protection circuit starts with PGOOD_DELAYED = LOW for ~300 ms,
  which holds ENABLE_GLOBAL = LOW → FET OFF → all downstream off.
  After 300 ms, PGOOD_DELAYED goes HIGH and normal operation can begin.

  IC: One gate of U_SCH — SN54SC6T17-SEP (hex Schmitt buffer, TSSOP-14)

  Circuit:
    3.3V (from Buck #1) ── [R_start: 1.0 MΩ, 1206] ── TIMING_A_NODE
    TIMING_A_NODE ─── [C_start: 330 nF, 1206 ceramic] ─── GND
    TIMING_A_NODE ─── IN of Schmitt buffer gate (one gate of SN54SC6T17-SEP)
    OUT of Schmitt buffer → PGOOD_DELAYED

  Timing calculation:
    t ≈ R × C × ln(VCC / (VCC − VT+))
    At typical VT+ = 1.20V, VCC = 3.3V: t = (1.0M)(330n) × ln(3.3/2.1) = 0.33 × 0.452 = 150 ms
    At min VT+ = 0.86V: t = 108 ms
    At max VT+ = 1.623V: t = 224 ms
    Range: ~110–225 ms. Adequate for PMIC startup sequences.
    If longer startup delay needed (e.g., for slow PMIC), increase C_start to 680 nF → range 225–460 ms.

  PGOOD_DELAYED = LOW during power-up (capacitor charging → input below VT+)
  PGOOD_DELAYED = HIGH after delay (cap charged → input above VT+) → normal op

────────────────────────────────────────────────────────────────────────────────
SECTION C: RETRY TIMER (~0.6–1.35 second)
────────────────────────────────────────────────────────────────────────────────

  PURPOSE: After a fault trips the system, hold it off for ~1 second to allow
  latchup current to dissipate and PMIC/buck capacitors to discharge before
  the next restart attempt.

  IMPLEMENTATION: RC charge + same SN54SC6T17-SEP Schmitt buffer chip.
  Uses second gate of U_SCH (same IC as startup delay section).

  This replaces the LM555CM/883C timer from v3.1 (datasheet unreadable,
  pin compatibility uncertain). The RC+Schmitt approach is equally reliable.

  Circuit:
    The capacitor TIMING_B_NODE is:
      Normally discharged to GND via Q_discharge (a small NPN BJT or
      an N-channel FET) when GLOBAL_FAULT is HIGH (no fault).
    When FAULT occurs (GLOBAL_FAULT → LOW):
      Discharge transistor turns OFF → 3.3V charges C_timer through R_timer.

    3.3V ─── [R_timer: 1.0 MΩ, 1206] ─── TIMING_B_NODE
    TIMING_B_NODE ─── [C_timer: 2.2 µF, 1206 ceramic X5R/X7R] ─── GND
    TIMING_B_NODE ─── IN of second Schmitt buffer gate
                       (gate #2 of U_SCH SN54SC6T17-SEP)

    Discharge path:
    Q_dis: MMBT2222A (NPN BJT, SOT-23, or similar small-signal transistor)
      Collector: TIMING_B_NODE
      Emitter: GND
      Base: via R_base (10 kΩ) from GLOBAL_FAULT signal

    When GLOBAL_FAULT = HIGH (3.3V) → Q_dis ON → collector low → TIMING_B = 0V → no charge
    When GLOBAL_FAULT = LOW (0V)   → Q_dis OFF → C_timer charges via R_timer

  Timer output (gate #2 of U_SCH):
    Input = TIMING_B_NODE, Output = RETRY_ACTIVE
    RETRY_ACTIVE = LOW while charging (below VT+) = system should stay OFF
    RETRY_ACTIVE = HIGH when charged above VT+ = one retry attempt

  Timing range (RC = 2.0 s, VCC = 3.3V):
    Shortest delay (min VT+ = 0.86V):  0.60 s
    Typical delay  (typ VT+ = 1.20V):  0.90 s
    Longest delay  (max VT+ = 1.623V): 1.35 s

  After RETRY_ACTIVE goes HIGH (above VT+), system attempts restart.
  If fault cleared → normal operation, Q_dis turns ON, discharges cap.
  If fault persists → fault signal immediately causes another timer cycle.
  This produces infinite retry with ~0.6–1.35 s between attempts.

  Q_dis radiation note: MMBT2222A is an NPN BJT (bipolar structure).
  Bipolar transistors are inherently radiation-tolerant; BJTs do not
  experience latchup by definition. TID may cause Hfe degradation at high
  dose, but at 30–50 krad, Hfe reduction is modest (<50%) and the circuit
  design is tolerant of this — any beta >10 is sufficient to switch the
  circuit. Gain margin is ample.

  U_SCH full connections (SN54SC6T17-SEP, TSSOP-14):
    Hex buffer; gates are: Y = A (non-inverting, Schmitt threshold)
    Gate 1: startup delay (TIMING_A → PGOOD_DELAYED)
    Gate 2: retry timer   (TIMING_B → RETRY_ACTIVE)
    Gates 3–6: unused; tie all inputs to GND

    Pinout: A1=1, Y1=2, A2=3, Y2=4, GND=5, Y3=6, A3=7 (TSSOP-14)
            VCC=14, Y4=13, A4=12, Y5=11, A5=10, Y6=9, A6=8

────────────────────────────────────────────────────────────────────────────────
SECTION D: ENABLE LOGIC
────────────────────────────────────────────────────────────────────────────────

  FINAL LOGIC:
    ENABLE_GLOBAL = PGOOD_DELAYED AND (NOT RETRY_ACTIVE)

  Explanation:
    PGOOD_DELAYED: HIGH after startup settling, LOW during startup
    RETRY_ACTIVE:  HIGH = retry in progress (keep FET off)
                   LOW  = OK to run
    NOT_RETRY:     Inverted RETRY_ACTIVE
    ENABLE_GLOBAL: HIGH = FET ON (system running)

  IC U_INV: SN54SC6T14-SEP (hex Schmitt inverter, TSSOP-14)
  IC U_AND: SN54SC4T08-SEP (quad 2-input AND gate, TSSOP-14)

  PART 1 — Invert RETRY_ACTIVE (one gate of U_INV):
    Same IC (U_INV) as master FET gate driver (Section 3).
    Use gate #1 of U_INV for FET gate drive (ENABLE_GLOBAL → INV_OUT → Q1 gate)
    Use gate #2 of U_INV for RETRY_ACTIVE inversion:
      Gate 2: IN = RETRY_ACTIVE, OUT = NOT_RETRY
    Gates 3–6: tie inputs to GND

  PART 2 — 2-input AND (one gate of U_AND):
    IC: SN54SC4T08-SEP (TSSOP-14)
    Gate 1:
      A = PGOOD_DELAYED
      B = NOT_RETRY
      Y = ENABLE_GLOBAL
    Gates 2–4: tie both inputs to GND (unused)

    SN54SC4T08-SEP pinout (TSSOP-14):
      Gate 1: A=1, B=2, Y=3
      Gate 2: A=4, B=5, Y=6
      GND=7
      Gate 3: Y=8, A=9, B=10
      Gate 4: Y=11, A=12, B=13
      VCC=14

  LOGIC TABLE:
  PGOOD_DLY  RETRY_ACTIVE  NOT_RETRY  ENABLE_GLOBAL  System State
     LOW          X            X           LOW         OFF (startup settling)
     HIGH        HIGH          LOW         LOW         OFF (retry in progress)
     HIGH         LOW         HIGH         HIGH        ON  (normal operation)

────────────────────────────────────────────────────────────────────────────────
POWER DISTRIBUTION FOR LOGIC
────────────────────────────────────────────────────────────────────────────────

  All logic ICs, comparators, and op-amps: powered at 3.3V from Buck #1 output.

  IMPORTANT: This means when master FET shuts off → Buck #1 loses 5V input
  → Buck #1 output collapses → protection circuit loses power.
  This resets the timer cap, comparators, and all logic to power-off state.
  The next attempt starts from a clean power-up condition.

  Consequence: FAULT signals go undefined for a brief moment (~10–50 ms) as
  protection circuit loses power. This is acceptable because the master FET
  is already OFF during this window; no false re-enable can occur because
  the AND gate also loses power (ENABLE_GLOBAL = LOW by default).

7. SYSTEM INTEGRATION AND SIGNAL FLOW
================================================================================

FAULT DETECTION AND SHUTDOWN SEQUENCE:

  1. Latchup occurs on a COTS component, e.g., the 1.2V rail.
  2. Load current on Rail 3 spikes from ~2A to >10A.
  3. V_sense_3 rises: 10A × 10mΩ = 100 mV.
  4. Difference amplifier (U_AMP_A, Amp C) outputs: 21.9 × 100mV = 2.19V.
  5. Comparator 3 (U_COMP_A Comp 3): V_AMP_3 = 2.19V > V_ref = 0.994V.
     → OUT driven LOW → FAULT_3 pulled LOW by open drain.
  6. OR cascade: FAULT_3 = LOW → OR gate output → GLOBAL_FAULT goes LOW.
  7. Q_dis base: GLOBAL_FAULT LOW → Q_dis turns OFF.
  8. C_timer begins charging from 0V through R_timer.
  9. GLOBAL_FAULT = LOW AND NOT_RETRY = ? — wait, RETRY_ACTIVE is still LOW
     (timer just started). So NOT_RETRY = HIGH. AND with PGOOD_DELAYED (HIGH)
     → ENABLE_GLOBAL = HIGH momentarily. The FET is still ON.

  Wait — there is a brief propagation window here. The enable AND logic
  is: ENABLE_GLOBAL = PGOOD_DELAYED AND NOT RETRY_ACTIVE. When RETRY_ACTIVE
  is still LOW (timer not yet fired), ENABLE_GLOBAL remains HIGH and FET
  stays ON. The system does not immediately shut off on fault detection alone.

  THE MISSING PIECE: A direct fault → FET OFF path is needed.

  CORRECTION TO LOGIC: ENABLE_GLOBAL must also go LOW on GLOBAL_FAULT.

  Revised logic: ENABLE_GLOBAL = PGOOD_DELAYED AND GLOBAL_FAULT AND NOT_RETRY_ACTIVE

  This requires a 3-input AND gate, OR cascading two 2-input AND gates:
    Stage 1: PGOOD_DELAYED AND GLOBAL_FAULT → PGOOD_AND_OK
    Stage 2: PGOOD_AND_OK AND NOT_RETRY_ACTIVE → ENABLE_GLOBAL

  Using U_AND (SN54SC4T08-SEP) which has 4 gates:
    Gate 1: PGOOD_DELAYED AND GLOBAL_FAULT → PGOOD_AND_OK
    Gate 2: PGOOD_AND_OK AND NOT_RETRY    → ENABLE_GLOBAL
    Gates 3, 4: unused, tie inputs to GND

  Updated logic table:
  PGOOD  GLOBAL_FAULT  RETRY_ACTIVE  ENABLE_GLOBAL  State
   LOW      X              X             LOW         Startup (off)
   HIGH     LOW            X             LOW         FAULT (shut off immediately)
   HIGH     HIGH           HIGH          LOW         Retry in progress (off)
   HIGH     HIGH           LOW           HIGH        Normal operation (on)

  This is the CORRECT behaviour: fault immediately cuts ENABLE_GLOBAL,
  simultaneously starting the retry timer, and keeping system off until
  both the fault clears AND the retry timer completes.

RETRY SEQUENCE:

  After FET turns OFF → Buck #1 power collapses → protection circuit loses
  power → timer capacitor discharges through R_timer (slowest, ~1 second to
  discharge through the same 1 MΩ resistor).
  NOTE: With Q_dis: when GLOBAL_FAULT goes LOW (which it will, since the
  latchup current disappears when power is cut), Q_dis turns ON and discharges
  C_timer rapidly through Q_dis. The timer then restarts clean on next power-up.

POWER-CYCLING ORDER:
  On shutdown: 5V_IN → FET OFF → Buck converters lose input → all rails
  collapse in <10 ms → latchup clears (no sustaining current).
  On restart: 5V_IN → FET ON → Buck converters start → PMIC startup →
  rails come up → protection circuit monitors from first moment.

9. PCB LAYOUT GUIDELINES
================================================================================

LAYER STACKUP (recommended 4-layer):
  Layer 1 (top):   components and signal traces
  Layer 2:         ground plane (solid pour, minimise splits)
  Layer 3:         power planes (5V_IN, 5V_OUT, 3.3V zones)
  Layer 4 (bottom): overflow routing and test points

PCB SIZE GUIDANCE:
  Target: ≤ 75 × 55 mm. TSSOP-14 footprint (~5 × 4.4 mm) is significantly
  smaller than SOIC-14 (~9 × 4 mm), recovering ~30% board area for logic ICs
  versus v3.1. CFP-14 (9.73 × 6.47 mm) for LMP7704-SP requires 2 large
  zones — these are the footprint-dominating components. Plan accordingly.

Q1 (BUP06CP038F-01, DPAK):
  - Place close to 5V input connector
  - DPAK has exposed drain pad on bottom; connect to copper pour
  - Minimum 6 thermal vias (0.3 mm) through drain pad to inner/bottom copper
  - 5V_IN trace to source: minimum 4 mm wide (carry 5A continuous)
  - 5V_OUT trace from drain: minimum 4 mm wide

LMP7704-SP (CFP-14, hermetic flatpack):
  - NOT compatible with standard SMD reflow profile
  - Requires hand soldering to pre-tinned pads, or a solderable socket
  - Coordinate land pattern and assembly method with PCB engineer
  - Hermetically sealed: no flux trapping concern, but inspect visually
  - Place both CFP-14 packages near shunt resistors to minimise sense traces

SHUNT RESISTORS (WSL2512):
  - Force+ and Force− pads on wide traces (carry full rail current)
  - Sense+ and Sense− pads connect only to amplifier inputs via thin traces
  - Kelvin connection: never mix Force and Sense pads
  - Sense trace pairs: matched length within 0.5 mm, routed together (50 mil spacing)
  - Shield sense traces from high-current paths with ground guard traces

COMPARATOR AND LOGIC (TSSOP-14):
  - Standard reflow compatible
  - 100 nF bypass cap within 2 mm of each VCC pin
  - Keep FAULT signal traces away from high-current shunt traces

RC TIMING NETWORKS:
  - Keep R_timer (1 MΩ) and C_timer (2.2 µF) away from high-impedance nodes
  - At 1 MΩ, PCB surface leakage can affect timing — use conformal coating
    or solder mask coverage around timing node pads
  - Do not run other traces near TIMING_A_NODE or TIMING_B_NODE

POWER PLANES:
  - Separate 5V_IN and 5V_OUT planes with FET between them
  - 3.3V zone fed from Buck #1 for protection circuit supply
  - Place 100 µF tantalum on both 5V_IN and 5V_OUT planes near Q1

SIGNAL INTEGRITY:
  - GLOBAL_FAULT and ENABLE_GLOBAL are critical signals; keep traces <5 cm
  - No stubs on FAULT signal lines
  - Pull-up resistors (10 kΩ) locate close to comparator outputs

================================================================================
11. TESTING PROCEDURES
================================================================================

STEP 1 — PRE-POWER CONTINUITY CHECKS (no power applied)
  1a. Measure resistance from 5V_IN to 5V_OUT: should be >10 kΩ (Q1 off, via R1)
  1b. Measure resistance ENABLE_GLOBAL to GND: should read as open drain pull-up
  1c. Verify all IC supply pins have continuity to correct rails
  1d. Verify no shorts: 5V to GND, 3.3V to GND, GATE to SOURCE of Q1

STEP 2 — FIRST POWER-ON (bench test, 5V 0.5A current limited)
  2a. Apply 5V with current limit 0.5A. Verify standby current < 100 mA.
  2b. Confirm PGOOD_DELAYED transitions HIGH after ~150–225 ms.
  2c. Confirm 5V_OUT appears after PGOOD_DELAYED goes HIGH.
  2d. Confirm 3.3V is present at op-amp and logic supply pins.

STEP 3 — INDIVIDUAL RAIL SENSE VERIFICATION (with resistive loads)
  For each rail n:
  3a. Apply rail voltage from bench supply. Set resistive load to rated current.
  3b. Measure V_AMP_n at op-amp output. Should read ~50–200 mV (below trip).
  3c. Increase load current to 150% of rated. V_AMP_n should read ~985–1005 mV.
      At this point FAULT_n should go LOW. Confirm with oscilloscope.
  3d. Verify GLOBAL_FAULT goes LOW when FAULT_n is LOW.
  3e. Verify 5V_OUT collapses within 20 µs of GLOBAL_FAULT going LOW.
      Measure with oscilloscope: probe trigger on FAULT_n falling edge;
      measure time to 5V_OUT reaching 50% of full voltage.

STEP 4 — RESPONSE TIME MEASUREMENT (critical)
  Use a power MOSFET + function generator to create a step-current fault.
  Do NOT use a potentiometer (gradual ramp is not representative of latchup).
  4a. Rails 1 & 2: verify response (FAULT → 5V_OUT collapse) ≤ 5 µs.
  4b. Rails 3–5:   verify response ≤ 8 µs.
  4c. Rail 6:      verify response ≤ 12 µs.

STEP 5 — RETRY TIMING
  5a. Create a sustained fault condition on any rail.
  5b. Measure the OFF period from 5V_OUT collapse to next restart attempt.
      Expected: 0.6 to 1.35 s. Record measured value.
  5c. Allow system to cycle 10 times. Confirm it never permanently latches.
  5d. Confirm each retry delay is within 0.6–1.35 s window.

STEP 6 — STARTUP DELAY VERIFICATION
  6a. Remove power completely. Re-apply.
  6b. Measure time from 5V_IN application to 5V_OUT appearing.
      Expected: 110–225 ms.
  6c. Repeat 5 times and record values.

STEP 7 — FAULT RECOVERY (self-clearing latchup simulation)
  7a. Apply a brief fault pulse (200 ms) then remove it.
  7b. Verify system shuts off, waits ~1 s, restarts, and recovers to normal.
  7c. Verify no permanent disable condition.

STEP 8 — POWER INTERRUPTION (OBC simulation)
  8a. Induce permanent fault (sustained overcurrent on any rail).
  8b. Observe system cycling for 5+ retry cycles.
  8c. Remove 5V_IN entirely.
  8d. Verify complete shutdown.
  8e. Re-apply 5V_IN. Verify normal startup sequence.

PASS/FAIL CRITERIA:
  Step 3: V_AMP_n within 200 mV of 1.0V at trip current ← allow tolerance
  Step 4: All response times within limits in Step 4 ← critical safety
  Step 5: Retry delay 0.5–1.5 s (wider than design spec, allows for test temp variation)
  Step 5: Infinite retry confirmed (no permanent latch after 10+ cycles)
  Step 6: Startup delay 80–300 ms

12. FAILURE MODE ANALYSIS
================================================================================

FAILURE 1: False trip during normal operation
  Cause: Noise on sense traces, inrush above 150%, op-amp oscillation
  Detection: System shuts off transiently; OBC observes brief gap
  Mitigation: 150% threshold provides 3× margin over steady-state noise
              Input filter capacitors (100 nF) on op-amp inputs help with HF noise
              Schmitt comparator hysteresis (if R_hyst installed) adds ~3 mV window
  Recovery: System retries in ~1 s; transient fault self-clears

FAILURE 2: Missed latchup (no detection)
  Cause: Response too slow, or current does not rise above 150%
  Most likely on Rail 6 (9.2 µs response). For a latchup that develops
  gradually rather than as a step, detection is delayed further.
  Mitigation: 150% threshold is conservative; most latchup events far exceed this
  Residual risk: Accept. Hardware-only detection cannot guarantee zero miss rate.

FAILURE 3: Protection circuit itself latches (CMOS logic SEL)
  Cause: An SEL in one of the SN54SC logic ICs or TLV1704 comparators
  All logic uses modern CMOS FinFET process. SEL LET threshold = 43 MeV·cm²/mg
  for SN54SC and TLV1704. LEO proton and HZE ion spectrum: most particles below
  this threshold. Risk is low but non-zero.
  Mitigation: If an SEL occurs in the protection circuit, ENABLE_GLOBAL may
  stay HIGH (FET stays on) or LOW (FET stays off permanently). Either is
  detectable by OBC. OBC should cut 5V_IN on detection of either:
  a) device unresponsive for >10 seconds, or
  b) device cycling without recovery
  LMP7704-SP (op-amps) are much more robust: LET = 85 MeV·cm²/mg and bipolar
  structure means SEL is essentially impossible.

FAILURE 4: Op-amp TID degradation over mission life
  LMP7704-SP rated 100 krad. Expected 3-year LEO dose at 400 km: ~9–45 krad
  (depending on shielding). Well within specification.
  Logic and comparators rated 30 krad — same dose range leaves marginal TID
  margin at mission end if shielding is minimal. Recommend minimum 2 mm Al
  shielding equivalent to reduce effective dose.

FAILURE 5: Stuck in infinite retry loop
  Cause: Permanent latchup that clears only partially, or component damage
  Detection: OBC monitors device; if device unresponsive for >15 seconds
  (i.e., >10 consecutive failed retries), OBC must cut 5V_IN from spacecraft bus
  Recovery: OBC commanded power restoration after ground analysis

FAILURE 6: Q1 thermal failure
  Cause: Sustained high current above rated capacity
  Mitigation: Q1 handles 5A continuously with 1.1W dissipation and
  1.3°C junction rise. Peak latchup current may reach 20–50A before
  FET turns off (<10 µs). Q1 rated -140A peak. Energy: 50A × 5V × 10µs = 2.5 mJ.
  Avalanche energy limit: 500 mJ. Well within spec.

13. OPEN ITEMS AND LIMITATIONS
================================================================================

OPEN ITEM 1: CFP-14 Assembly Process (BLOCKING for PCB layout finalization)
  The LMP7704-SP CFP-14 package requires a non-standard assembly process.
  PCB engineer must confirm: (a) land pattern dimensions, (b) whether hand
  soldering or socket is used, (c) impact on PCB area and layer stack.
  This decision must be made before Gerber file generation.

OPEN ITEM 2: Buck #1 Availability at Power-Up
  Protection circuit draws 3.3V from Buck #1. Buck #1 itself is protected
  by the master FET. This creates a bootstrapping dependency: Buck #1 needs
  to start for protection logic to work, but protection logic controls whether
  Buck #1 gets power. Resolution: the startup delay circuit (Section 6B)
  handles this correctly — capacitor TIMING_A charges from 3.3V (Buck #1),
  and ENABLE_GLOBAL stays LOW during this time. If Buck #1 fails to start
  (not latchup-related), the system remains off safely. Document this
  dependency and test on bench to confirm no startup race condition.

OPEN ITEM 3: Confirm ARM Cortex-A72 Process Node
  The AI cores' process node is unknown without silicon identification.
  FinFET (16nm/14nm) is more latchup-resistant than 28nm planar. If AI cores
  are FinFET, the 150% threshold may be conservative to the point where a
  real latchup event never exceeds it. Recommend verifying latchup test data
  for specific die if available from manufacturer.

OPEN ITEM 4: Rail 4 Voltage Drop Budget
  100 mΩ shunt on 1.35V CPU rail causes 40 mV drop at rated current.
  PCB trace resistance must be minimised for this rail. After layout, perform
  voltage budget analysis: shunt drop + trace drop + connector drop < 67.5 mV
  (5% of 1.35V). If PCB trace resistance exceeds 27.5 mΩ (= 27.5 mV at 1A),
  the CPU may see a voltage below specification. Verify with powered test.

LIMITATION 1: No per-rail isolation
  All rails are cut simultaneously when any one faults. A latchup on Rail 4
  (CPU low-voltage rail, 400 mA) cuts the AI core 3.3V rails unnecessarily.
  This is intentional: with the PMIC present, individual rail switching would
  require individual FETs per output of the PMIC — impractical on a space-
  constrained board. The OBC observes which FAULT signal triggered for diagnostics.

LIMITATION 2: Micro-SEL not detected
  Any latchup event that resolves within ~10 µs will not trip the protection
  circuit. This is acceptable: micro-SEL events are self-quenching in modern
  CMOS and do not cause permanent damage at low currents.

LIMITATION 3: Retry delay varies 2:1 (0.6–1.35 s)
  The Schmitt trigger threshold tolerance is wide in the SN54SC6T17-SEP.
  This means retry delay varies by a factor of 2.25 across the TID-degraded
  threshold range. For the mission purpose (giving PMIC time to discharge
  and reset), any delay >0.5 s is adequate. This limitation is accepted.
