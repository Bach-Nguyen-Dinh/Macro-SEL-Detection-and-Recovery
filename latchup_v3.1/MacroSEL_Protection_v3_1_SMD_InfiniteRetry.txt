1. SYSTEM ARCHITECTURE OVERVIEW
================================================================================

PROTECTION STRATEGY:
- 1 master P-channel MOSFET switch (high-side, 5V input)
- 6 independent current sensors (one per rail, for diagnostics)
- Any fault → all rails shut down together
- Hardware-only auto-retry: INFINITE attempts, 1 second between attempts
- 300ms startup delay from PMIC_PGOOD
- RC snubber gate driver for clean, reliable switching
- OBC monitors device and cuts power if stuck in retry loop

PERFORMANCE:
Response Time: 10-18µs (well within 100µs macro-SEL requirement)
Retry Delay: 1.0 second (reduced component stress)
Retry Limit: INFINITE (OBC monitors and cuts power if needed)
PCB AREA: ~4,125 mm² (75mm × 55mm) - 15% SMALLER than v3.0
Cost: ~$806 (vs $850 in v3.0) - $42 savings

OPERATIONAL CONCEPT:
1. Latchup detected → System shuts down in <20µs
2. System stays off for 1 second
3. System attempts restart
4. If latchup cleared → Normal operation resumes
5. If latchup persists → Repeats steps 2-3 indefinitely
6. OBC monitors device responsiveness
7. OBC cuts power if device stuck in retry loop

SECTION A: STARTUP DELAY (300ms) - UNCHANGED FROM v3.0
------------------------------------------------------

Prevents false trips during power-up inrush.

SMD Components:
- R11: 1MΩ, 1206 SMD
- C5: 330nF, 1206 SMD ceramic
- U3: CD4093BM/883B (SOIC-14, Schmitt trigger)

Circuit:

    PMIC_PGOOD ────┬─────[R11: 1MΩ]─────┬──→ Schmitt IN
                   │                     │
                   │                  [C5: 330nF]
                   │                     │
                   └─────────────────────┴──→ GND

Time constant: τ = 1MΩ × 330nF = 330ms

Connections:

**R11 (1MΩ, 1206 SMD):**
- Pad 1: PMIC_PGOOD signal (from PF8100 PMIC)
- Pad 2: DELAY_NODE

**C5 (330nF, 1206 SMD):**
- Pad 1: DELAY_NODE (same as R11 pad 2)
- Pad 2: GND via

**U3 (CD4093BM/883B SOIC-14 Schmitt trigger):**

CD4093 pinout (SOIC-14):
     ┌─────┴─────┐
IN1A │1       14│ VCC
IN1B │2       13│ IN4B
OUT1 │3       12│ IN4A
OUT2 │4       11│ OUT4
IN2A │5       10│ OUT3
IN2B │6        9│ IN3B
 GND │7        8│ IN3A
     └───────────┘

Use gate #1 (pins 1, 2, 3):
- Pin 1 (IN1A): Connect to DELAY_NODE
- Pin 2 (IN1B): Connect to VCC (+5V) - makes this a buffer
- Pin 3 (OUT1): This is **PGOOD_DELAYED** signal
- Pin 14 (VCC): +5V
- Pin 7 (GND): Via to ground plane
- 0.1µF (1206) cap from pin 14 to pin 7, <2mm from IC

Operation:
- PMIC powers up → PGOOD goes HIGH
- C5 charges through R11 slowly
- After 330ms, voltage crosses Schmitt threshold (~2.5V)
- OUT1 (pin 3) goes HIGH = PGOOD_DELAYED
- System can now enable


SECTION B: FAULT AGGREGATION (6-input OR) - UNCHANGED FROM v3.0
---------------------------------------------------------------

Combines 6 FAULT signals into one GLOBAL_FAULT.

SMD Components:
- U2: Two CD4072BM/883B ICs (SOIC-14, dual 4-input OR gates)

CD4072 pinout (SOIC-14):
     ┌─────┴─────┐
 IN1A│1       14│ VCC
 IN2A│2       13│ IN1B
 IN3A│3       12│ IN2B
 IN4A│4       11│ IN3B
OUTA │5       10│ IN4B
OUTB │6        9│ OUTB
 GND │7        8│ NC
     └───────────┘

**IC #1 (First CD4072BM):**
- Pin 14: +5V
- Pin 7: GND via
- 0.1µF (1206) bypass cap

OR gate A (handle FAULT_1 through FAULT_4):
- Pin 1: FAULT_1 signal (from comparator #1)
- Pin 2: FAULT_2 signal (from comparator #2)
- Pin 3: FAULT_3 signal (from comparator #3)
- Pin 4: FAULT_4 signal (from comparator #4)
- Pin 5: Output = OR_AB (intermediate signal)

OR gate B (combine OR_AB with OR_CD):
- Pin 12: OR_AB (from pin 5 same IC)
- Pin 11: OR_CD (from second IC pin 5)
- Pin 13: Tie to GND (unused input)
- Pin 10: Tie to GND (unused input)
- Pin 6: Output = **GLOBAL_FAULT** signal

**IC #2 (Second CD4072BM):**
- Pin 14: +5V
- Pin 7: GND via
- 0.1µF (1206) bypass cap

OR gate A (handle FAULT_5 and FAULT_6):
- Pin 1: FAULT_5 signal (from comparator #5)
- Pin 2: FAULT_6 signal (from comparator #6)
- Pin 3: Tie to GND (unused)
- Pin 4: Tie to GND (unused)
- Pin 5: Output = OR_CD

Logic:
- All FAULT_n = HIGH → OR_AB = HIGH, OR_CD = HIGH → GLOBAL_FAULT = HIGH ✓
- Any FAULT_n = LOW → GLOBAL_FAULT = LOW (fault detected)


SECTION C: RETRY TIMER (1 SECOND DELAY) - MODIFIED FROM v3.0
-------------------------------------------------------------

When fault detected, generate 1 second pulse (system shutdown period).

**CHANGED:** Timer delay increased from 100ms to 1 second

SMD Components:
- U4: LM555CM/883C (SOIC-8, timer)
- R12: **10MΩ, 1206 SMD** (CHANGED from 1MΩ in v3.0)
- C6: 100nF, 1206 SMD
- C7: 0.1µF, 1206 SMD (decoupling)

LM555 pinout (SOIC-8):
     ┌─────┴─────┐
  GND│1        8│ VCC
 TRIG│2        7│ DISCH
  OUT│3        6│ THRESH
RESET│4        5│ CTRL
     └───────────┘

Configuration: Monostable (one-shot) mode

Connections:

**U4 Power:**
- Pin 8 (VCC): +5V
- Pin 1 (GND): Via to ground plane
- C7 (0.1µF, 1206): Between pins 8 and 1, <2mm from IC

**Timing network:**
- Pin 7 (DISCH) connect to pin 6 (THRESH) → TIMING_NODE

**R12 (10MΩ, 1206 SMD) - CRITICAL CHANGE:**
- Pad 1: VCC (+5V)
- Pad 2: TIMING_NODE (pins 6 and 7)

**C6 (100nF, 1206 SMD):**
- Pad 1: TIMING_NODE
- Pad 2: GND via

Time delay calculation:
t = 1.1 × R × C = 1.1 × 10MΩ × 100nF = 1.1 seconds ≈ **1.0 second** ✓

NOTE: 10MΩ is a high value. If not available in 1206:
- Alternative 1: Use 2× 5.1MΩ in series (1206 each)
- Alternative 2: Use 1MΩ + 1µF capacitor (t = 1.1 × 1MΩ × 1µF = 1.1s)
- Alternative 3: Use 2010 size for 10MΩ resistor (larger package)

**Trigger input:**
- Pin 2 (TRIG): Connect GLOBAL_FAULT directly
- When GLOBAL_FAULT goes LOW → timer triggers

**Control:**
- Pin 5 (CTRL): Optional 10nF cap to GND (noise filtering)

**Reset:**
- Pin 4 (RESET): Tie to VCC (always enabled)
- Or connect to PGOOD_DELAYED (disable timer during startup)

**Output:**
- Pin 3 (OUT): This is **RETRY_PULSE** signal
- HIGH for ~1.0 second when triggered
- LOW otherwise

Operation:
- GLOBAL_FAULT goes LOW (any rail faulted)
- Timer triggers → OUT (pin 3) goes HIGH for 1 second
- During this time, system shut down (ENABLE_GLOBAL = LOW)
- After 1 second, OUT returns LOW → system attempts restart
- If fault persists → cycle repeats indefinitely


SECTION D: REMOVED - Counter and Latch
--------------------------------------

**THESE SECTIONS NO LONGER EXIST IN v3.1:**
- Counter (was CD4017, counted retry attempts)
- Permanent latch (was CD4013, disabled after 5 retries)
- Power-on reset circuit (was 10kΩ + 10µF, reset counter/latch)

System now retries forever - OBC monitors and cuts power if needed.


SECTION E: SIMPLIFIED ENABLE GENERATION (2-INPUT AND)
-----------------------------------------------------

**SIMPLIFIED:** Enable logic now only 2 inputs (was 3 inputs in v3.0)

Final logic: **ENABLE_GLOBAL = PGOOD_DELAYED AND (NOT RETRY_PULSE)**

Where:
- PGOOD_DELAYED: From Section A (startup delay complete)
- RETRY_PULSE: From Section C (retry in progress)
- NO MORE LATCH_OFF (removed in v3.1)

SMD Components:
- U7: CD4081BM/883B (SOIC-14, quad AND gate) - only need 1 gate
- U8: CD4049UM/883B (SOIC-16, hex inverter) - only need 1 gate

**1. Invert RETRY_PULSE:**

Using U8 (CD4049UM SOIC-16 inverter):

     ┌─────┴─────┐
  IN1│1       16│ VCC

- Pin 1 (IN1): Connect to RETRY_PULSE signal
- Pin 2 (OUT1): Output is **NOT_RETRY** signal
- Pins 7 AND 16 (both VCC): Connect to +5V
- Pin 8 (GND): Via to ground plane
- 0.1µF (1206) bypass cap between pin 16 and pin 8
- Unused inputs (pins 3,5,9,11,13,15): Tie to GND or leave floating
- Unused outputs: Leave unconnected

**2. Build 2-input AND:**

Using U7 (CD4081BM SOIC-14 AND gate):

     ┌─────┴─────┐

- Pin 7 (GND): Via to ground plane
- 0.1µF (1206) bypass cap from pin 14 to pin 7

**3. Connect ENABLE_GLOBAL to master switch:**
- ENABLE_GLOBAL signal goes to Section 3 (master switch)
- Connects to U1 (master inverter) pin 1 input

Logic table (SIMPLIFIED from v3.0):

| PGOOD_DELAYED | RETRY_PULSE | NOT_RETRY | ENABLE_GLOBAL | System State      |
|---------------|-------------|-----------|---------------|-------------------|
| LOW (startup) | X           | X         | LOW           | OFF (waiting)     |
| HIGH          | HIGH        | LOW       | LOW           | OFF (retrying)    |
| HIGH          | LOW         | HIGH      | **HIGH**      | **ON (running)**  |

Note: No more LATCH_OFF condition - system never permanently disables!


POWER DISTRIBUTION FOR LOGIC ICs:
---------------------------------

All logic ICs powered from +5V OUTPUT (protected 5V, after master switch).

Why? If master switch turns OFF, logic loses power:
- Prevents gate drive during shutdown
- Clean power-up on retry
- No contention or floating signals

Each SOIC IC requires:
- VCC pin(s): 0.5mm trace to +5V OUTPUT
- GND pin(s): Via directly to ground plane
- 0.1µF (1206) bypass cap <2mm from IC


COMPLETE SIGNAL FLOW (v3.1 SIMPLIFIED):
---------------------------------------

STARTUP:
1. PMIC powers up → PMIC_PGOOD = HIGH
2. RC delay (R11/C5) charges for 330ms
3. Schmitt trigger (U3) outputs PGOOD_DELAYED = HIGH
4. If no faults: RETRY_PULSE = LOW
5. Inverter (U8): NOT_RETRY = HIGH
6. AND gate (U7): PGOOD_DELAYED AND NOT_RETRY = HIGH
7. ENABLE_GLOBAL = HIGH
8. Master inverter (U1): ENABLE_GLOBAL → Gate LOW → FET ON
9. System runs

FAULT DETECTION & RETRY:
1. Latchup on any rail → FAULT_n = LOW
2. OR gates (U2) → GLOBAL_FAULT = LOW
3. Timer (U4) triggers → RETRY_PULSE = HIGH (1 second)
4. Inverter (U8): NOT_RETRY = LOW
5. AND gate (U7): PGOOD_DELAYED AND NOT_RETRY = LOW
6. ENABLE_GLOBAL = LOW
7. Master inverter (U1): ENABLE_GLOBAL → Gate HIGH → FET OFF
8. System shuts down (5V_OUTPUT = 0V)
9. After 1 second: RETRY_PULSE returns LOW
10. ENABLE_GLOBAL = HIGH → System restarts
11. If latchup cleared → normal operation
12. If latchup persists → repeat from step 1 (INFINITE RETRY)

OBC MONITORING:
- OBC monitors device responsiveness (OS heartbeat, network ping, etc.)
- If device stuck in retry loop (not responding for X seconds)
- OBC cuts 5V INPUT power to entire device
- Manual intervention or ground command required to restore power

FUNCTIONAL TESTING (v3.1 DIFFERENCES):

11. FAILURE MODE ANALYSIS
================================================================================

FAILURE MODES (v3.1 SPECIFIC):
------------------------------

**NEW FAILURE MODE: Stuck in Infinite Retry Loop**
- Symptom: System cycles ON/OFF forever (1 second period)
- Cause: Permanent latchup or damaged component
- Detection: OBC monitors device unresponsive >10-30 seconds
- Mitigation: OBC cuts 5V INPUT power to entire device
- Prevention: None (this is expected behavior for permanent faults)
- Recovery: Ground command to restore power, investigate cause

**IMPROVED: No False Permanent Lockout**
- v3.0 could permanently disable after 5 false trips
- v3.1 keeps trying, so transient faults always recover
- Benefit: More robust recovery from temporary issues

**CONSIDERATION: Component Wear from Cycling**
If device stuck in retry loop for extended period:
- Buck converter stress: ~1 power cycle per second
- PMIC stress: Repeated soft-start sequences
- P-FET stress: Thermal cycling every second
- Tantalum caps: Inrush current cycling

However:
- OBC monitoring prevents extended cycling
- Most macro-SEL events are transient (clear within seconds)
- 1 second delay significantly reduces stress vs 100ms
